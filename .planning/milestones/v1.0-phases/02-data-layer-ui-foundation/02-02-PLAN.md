  <name>Task 2: CSS spinner in index.html and boot integration in App.tsx</name>
  <files>

  <action>
    **Step 1: Add CSS spinner to index.html**

    In `app/index.html`, add a CSS spinner inside the `#root` div. This spinner appears during the pre-React window (~200-800ms while JS loads and React hydrates). Once React mounts, it replaces the entire `#root` content, so the spinner disappears automatically.

    The spinner should be:
    - A simple CSS-only spinner (no JS) — a rotating circle/ring
    - Centered in the viewport (flexbox center)
    - Uses the same color scheme as the app (respects dark mode via the existing anti-flash script that adds `.dark` class)
    - Styles defined inline in a `<style>` tag within `#root` or in the `<head>` — NOT in index.css (must work before any CSS bundling)
    - Keep it minimal: small rotating ring, no text

    Example structure inside `<div id="root">`:
    ```html
    <div style="display:flex;align-items:center;justify-content:center;min-height:100vh">
      <div class="sp-spinner"></div>
    </div>
    ```
    With a `<style>` block in `<head>` for the spinner animation (or inline styles).

    **Step 2: Wire boot into App.tsx**

    Modify `app/src/App.tsx` to:
    1. Import `bootMetadata` from `@/lib/metadata`
    2. Import `useAppStore` from `@/stores/app-store`
    3. Import `ContentSkeleton`, `ErrorState` from `@/components/loading`
    4. Add `useEffect(() => { bootMetadata() }, [])` to trigger metadata load on app mount
    5. Get `status` from `useAppStore` using a scalar selector: `const status = useAppStore(s => s.status)`
    6. Render conditionally based on status:
       - `'idle'` or `'loading'`: render `<ContentSkeleton />` in place of `<Outlet />`
       - `'error'`: render `<ErrorState />` in place of `<Outlet />`
       - `'ready'`: render `<Outlet />` (the actual page content)
    7. The `<Header />` renders ALWAYS regardless of status — per user decision "Header with navigation renders immediately on first paint"
    8. Add a subtle fade-in transition (~200ms) when transitioning from skeleton to real content. Use a CSS class or inline style with `transition: opacity 200ms ease` and conditional opacity. OR use a simple approach: wrap the `<Outlet />` in a div with `animate-in fade-in duration-200` (from tw-animate-css which is already installed).

    The layout structure should remain:
    ```tsx
    <div className="flex min-h-screen flex-col bg-background text-foreground">
      <Header />
      <main className="flex flex-1 flex-col pt-14">
        {status === 'ready' ? <Outlet /> : status === 'error' ? <ErrorState /> : <ContentSkeleton />}
      </main>
    </div>
    ```

    IMPORTANT: The `bootMetadata()` function must only be called once. Use an empty dependency array `[]` in useEffect. The function itself is idempotent (checking status before proceeding), but avoid double-calls.
  </action>
