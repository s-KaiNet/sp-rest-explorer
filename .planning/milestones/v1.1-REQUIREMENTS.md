# Requirements Archive: v1.1 Search, Types & Polish

**Archived:** 2026-02-15
**Status:** All 13 requirements validated

## v1.1 Requirements

### Search

- [x] **SRCH-01**: User can open command palette with Cmd+K / Ctrl+K keyboard shortcut — validated (Phase 6)
- [x] **SRCH-02**: User can type a query and see matching entities, functions, and properties in real-time — validated (Phase 6, refined in 07.1)
- [x] **SRCH-03**: User can select a search result and navigate directly to that item's detail view — validated (Phase 6, fixed in 07.1 with pre-computed paths)
- [x] **SRCH-04**: Search results are grouped by kind (entities, functions, properties) for scannability — validated (Phase 6, restructured to 2 groups in 07.1: Entities + API Endpoints)

### Types

- [x] **TYPE-01**: User can browse a flat filterable list of all complex types and enum types in the sidebar — validated (Phase 7, gap closure in 07-03 to include all types)
- [x] **TYPE-02**: User can click a type to see its full definition (properties for complex types, members for enums) — validated (Phase 7)
- [x] **TYPE-03**: User can see the inheritance chain for a type (base type and derived types) — validated (Phase 7, derived types added in 07-03)
- [x] **TYPE-04**: User can see which entities and functions reference a type (precomputed used-by index) — validated (Phase 7, O(1) lookup via precomputed index)
- [x] **TYPE-05**: User can click any type reference in Explore API views to jump to its detail in Explore Types — validated (Phase 7, TypeLink updated for complex type awareness)
- [x] **TYPE-06**: User can navigate to Explore Types from the main header navigation — validated (Phase 7)

### Navigation

- [x] **NAV-03**: User can copy the _api/... path to clipboard via a button in the breadcrumb bar — validated (Phase 8, plan 08-02)

### Content

- [x] **INFO-01**: User can view a How It Works page explaining where metadata comes from and how the app works — validated (Phase 8, plan 08-01)
- [x] **INFO-02**: User can navigate to How It Works from the home screen or header — validated (Phase 8, plan 08-01)
