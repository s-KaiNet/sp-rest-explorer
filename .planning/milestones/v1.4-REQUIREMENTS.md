## v1.4 Requirements — Unify Icons

### Icon System

- [x] **ICON-01**: User sees distinct Lucide icons for each of the 4 API types (root, nav property, function, type/entity) — *validated v1.4, Phase 13 (13-01)*
- [x] **ICON-02**: A reusable TypeIcon component renders the correct icon and color given a kind — *validated v1.4, Phase 13 (13-01, 13-02)*
- [x] **ICON-03**: Root endpoints use a dedicated `--type-root` CSS color token (green, OKLCH hue 155) — *validated v1.4, Phase 13 (13-01, 13-02)*
- [x] **ICON-04**: Types/entities use an orange/amber `--type-entity` CSS color token (OKLCH hue ~75-85) distinct from root green — *validated v1.4, Phase 13 (13-01)*

### Explore API

- [x] **EAPI-01**: Explore API sidebar displays type icons to the left of entry labels (`[icon] Label`) — *validated v1.4, Phase 14 (14-01)*
- [x] **EAPI-02**: Explore API sidebar uses Lucide icons instead of `FN`, `NAV`, `<>` text badges — *validated v1.4, Phase 14 (14-01)*
- [x] **EAPI-03**: Explore API welcome screen uses updated Lucide icon with correct type color — *validated v1.4, Phase 14 (14-01)*
- [x] **EAPI-04**: Root-level namespace-grouped entries show type icons consistent with the new icon system — *validated v1.4, Phase 14 (14-01)*

### Cross-View Consistency

- [x] **XVEW-01**: Cmd+K search modal results display Lucide icons instead of `<>`, `ƒ`, `NAV` text symbols — *validated v1.4, Phase 15 (15-01)*
- [x] **XVEW-02**: "Root" pill badge is removed from individual search result items — *validated v1.4, Phase 15 (15-01)*
- [x] **XVEW-03**: Home page recently visited cards display Lucide icons matching the new icon system — *validated v1.4, Phase 15 (15-01)*
- [x] **XVEW-04**: Explore Types welcome screen uses updated Lucide icon with correct type color — *validated v1.4, Phase 15 (15-02)*
- [x] **XVEW-05**: Explore Types sidebar uses Lucide icons consistent with the new icon system — *validated v1.4, Phase 15 (15-02)*

### Entity Link Colors

- [x] **LINK-01**: Entity type links (e.g., SP.Alert, SP.Group) render in the entity type color (orange/amber `--type-entity`) instead of hardcoded emerald green — *validated v1.4, Phase 16 (16-01)*
- [x] **LINK-02**: Collection wrapper links (e.g., "Collection" prefix) remain muted-foreground, only the inner type name uses entity color — *validated v1.4, Phase 16 (16-01)*

### Search Modal Layout

- [x] **SMOD-01**: Search modal footer hint bar shows "Navigate" on the left and "Open" + "Close" on the right — *validated v1.4, Phase 17 (17-01)*
