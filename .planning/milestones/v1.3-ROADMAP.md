# Milestone v1.3: Improvements

**Status:** ✅ SHIPPED 2026-02-17
**Phases:** 11-12
**Total Plans:** 4

## Overview

Fixed search UX issues (literal dot matching, group collapse stability, path-length sort, hover feedback), corrected nullable property display logic, and relocated breadcrumb to content area with scroll isolation. Targeted fixes that improved daily usability of existing features.

## Phases

### Phase 11: Search UX Fixes ✅ (2026-02-17)

**Goal**: Search results in the Cmd+K palette behave predictably — dots match literally, groups collapse smoothly, results sort sensibly, and items feel interactive
**Depends on**: Phase 07.2 (MiniSearch index, path search, CommandPalette component)

**Requirements:** SRCH-05, SRCH-06, SRCH-07, SRCH-08

**Success Criteria:**
1. Typing "SP.File" in Cmd+K returns only results containing the literal substring "SP.File" — not "SP" and "File" as separate token matches
2. Clicking anywhere on a search result group header toggles collapse; collapsing/expanding does not cause the header text to shift position
3. API Endpoints results appear sorted by path length (shortest first) when a search query is active
4. Hovering over any search result item shows a visible background highlight and the cursor changes to pointer

Plans:
- [x] 11-01: Literal substring matching, collapsible group headers, path-length sorting, hover feedback
- [x] 11-02: Fix invisible hover/selected colors with theme-resilient bg-foreground/8 overlays (gap closure)

**Details:**
- 11-01: Added `literalNameSearch()` bypass for queries with special characters (dots, parens) — skips MiniSearch, uses case-insensitive substring matching. Redesigned `SearchGroup` with standalone header div above headingless `CommandGroup` for layout-stable collapse. Endpoints sorted by path character length. Hover feedback with `cursor-pointer`, `hover:bg-accent`, and `→` arrow hint. 3 files modified.
- 11-02: Dark mode `--accent` color was identical to `--popover` (both `oklch(0.22 0.01 260)`), making `hover:bg-accent` and `data-[selected=true]:bg-accent` invisible. Replaced with `bg-foreground/8` semi-transparent overlays on both CommandPalette items and CommandItem base. 2 files modified.

### Phase 12: Detail & Layout Fixes ✅ (2026-02-17)

**Goal**: Entity property nullable display is accurate and the Explore API breadcrumb sits in the correct visual region with proper scroll isolation
**Depends on**: None (PropertiesTable and BreadcrumbBar exist from v1.0/v1.1)

**Requirements:** DETL-01, LAYT-01

**Success Criteria:**
1. On any entity properties table, a property with `"nullable": false` in metadata shows "no" in the Nullable column; all other properties show "yes"
2. The Explore API breadcrumb trail renders inside the main content area (below the header chrome), not within the top header bar, with scrollbar only covering content below the breadcrumb

Plans:
- [x] 12-01: Nullable column strict equality fix and breadcrumb repositioned into content area
- [x] 12-02: Split content area so scrollbar excludes breadcrumb — flex-column layout (gap closure)

**Details:**
- 12-01: Changed nullable column from truthy check (`prop.nullable ? ...`) to strict equality (`prop.nullable === false ? "no" : "yes"`). Moved BreadcrumbBar from full-width position above sidebar into content area, conditional on `!isRoot && segments.length > 0`. Added sticky positioning with bottom border. 3 files modified.
- 12-02: UAT revealed sticky breadcrumb caused transparency artifacts and scrollbar spanning both breadcrumb and content. Restructured content panel to flex column: breadcrumb as non-scrolling `shrink-0` child above dedicated `overflow-y-auto` scroll wrapper. Removed `sticky top-0 z-10` from BreadcrumbBar. 2 files modified.

## Milestone Summary

**Key Decisions:**
- Bypass MiniSearch entirely for special-char queries rather than modifying the tokenizer — preserves fuzzy/prefix behavior for simple queries
- Render SearchGroup header as plain div above headingless CommandGroup — maintains cmdk keyboard navigation while giving full control over header layout
- Show-more button as plain div (not CommandItem) to avoid cmdk interference
- Semi-transparent foreground overlays (`bg-foreground/8`) instead of semantic accent tokens when accent matches background — theme-resilient
- Strict equality (`=== false`) for optional boolean fields with "default true" semantics
- Flex column split pattern (overflow-hidden parent + overflow-y-auto child) instead of sticky positioning — structurally prevents scroll artifacts

**Issues Resolved:**
- MiniSearch tokenizer splitting dots in queries like "SP.File" into separate token matches (literalNameSearch bypass)
- Layout shift when collapsing/expanding search result groups (standalone header div above headingless CommandGroup)
- Invisible hover/selection highlight in dark mode (accent color identical to popover background)
- Nullable column showing "no" for properties with missing/undefined nullable metadata
- Breadcrumb transparency artifacts and scrollbar overlap on scroll (flex-column split layout)

**Issues Deferred:**
- CHLG-01 through CHLG-06: API Changelog view (deferred to future milestone)
- ADDL-02: GitHub Actions CI/CD auto-deployment (deferred to future milestone)

**Technical Debt Incurred:**
- None — v1.3 was a fix/improvement milestone that resolved issues rather than creating debt
