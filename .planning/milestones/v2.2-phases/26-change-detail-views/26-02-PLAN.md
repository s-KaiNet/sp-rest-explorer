**Replace the placeholder block** (the `{totalChanges > 0 && (...)}` block containing "Detailed change views coming in a future update.") with:

```tsx
{totalChanges > 0 && diff && (
  <div className="mt-8 space-y-2">
    {/* Root Functions section — before entities per user decision */}
    <CollapsibleSection
      id="root-functions"
      title="Root Functions"
      count={diff.functions.length}
      emptyMessage="No root function changes"
    >
      <RootFunctionsTable functions={diff.functions} />
    </CollapsibleSection>

    {/* Entities section */}
    <CollapsibleSection
      id="entities"
      title="Entities"
      count={diff.entities.length}
      emptyMessage="No entity changes"
    >
      <div className="space-y-3 pt-2">
        {[...diff.entities]
          .sort((a, b) => a.name.localeCompare(b.name))
          .map((entity) => (
            <EntityChangeCard key={entity.name} entity={entity} />
          ))}
      </div>
    </CollapsibleSection>
  </div>
)}
```

**Key points:**
- Root functions section comes FIRST (before entities) per user decision
- Both sections wrapped in CollapsibleSection with count headers per user decision
- Entities sorted alphabetically per user decision (one list, not grouped by change type)
- The `diff &&` guard ensures diff is non-null before accessing .functions/.entities
- `space-y-2` between the two top-level sections keeps them visually distinct
- `space-y-3 pt-2` between entity cards provides breathing room
