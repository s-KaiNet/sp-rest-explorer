<task type="auto">
  <name>Task 2: Port DiffGenerator as compute-diff and transform-delta modules</name>
  <files>
    app/src/lib/diff/compute-diff.ts

  <action>
1. Create `app/src/lib/diff/compute-diff.ts` — porting from `az-funcs/src/diffGenerator.ts`:
   - Export `computeRawDiff(current: Metadata, previous: Metadata): object | undefined`
   - Use ESM import: `import { create } from 'jsondiffpatch'` (NOT `require`)
   - Create jsondiffpatch instance with:
     - `objectHash: (obj: any) => obj.id || obj.name || obj._id || '$$index:' + index`
     - `propertyFilter: (name: string) => !name.startsWith('SP.Data.')`
   - Port `fixJson()` as a private function `prepareForDiff(metadata: Metadata): Metadata`:
     - Deep-clone with `structuredClone()` (NOT `JSON.parse(JSON.stringify())` — modern API, cleaner)
     - Re-key functions by `getUniqueFunctionName()` composite key
     - Replace entity `functionIds` arrays with expanded function objects (same logic as legacy)
   - Port `getUniqueFunctionName(func)` as a private function preserving exact composite key format: `${parentObject}-${name}-${returnType}`
   - Return `jsdiff.diff(preparedPrevious, preparedCurrent)` — note: previous first, current second (so adds = things in current not in previous)
   - Return `undefined` if no differences (jsondiffpatch returns undefined for identical inputs)

2. Create `app/src/lib/diff/transform-delta.ts` — porting delta-to-DiffChanges logic from `az-funcs/src/templateGenerator.ts`:
   - Export `transformDelta(delta: any, currentMetadata: Metadata): DiffChanges`
   - Port `detectChangeType(node: any[]): ChangeType` as private function:
     - `node.length === 1` → `'added'`
     - `node.length === 3 && node[1] === 0 && node[2] === 0` → `'removed'`
     - `node.length === 2` → `'updated'`
   - For entities loop: iterate `delta.entities`, for each:
     - If value is an Array → entity added/removed (detectChangeType)
     - If added: copy properties/navigationProperties/functionIds from the added entity object (index 0 of the array) with changeType 'added'
     - If value is an Object → entity updated: call `extractChildChanges()` for 'properties', 'navigationProperties', 'functionIds'
   - Port `populateChildProperties` as `extractChildChanges(entityDelta: any, key: string): DiffPropertyChange[]`:
     - Check `entityDelta[key]` exists, is object, has `_t === 'a'` (jsondiffpatch array marker)
     - For each non-'_t' key, if value is Array: extract name, typeName (or returnType), detectChangeType
   - For functions loop: iterate `delta.functions`, only process keys starting with `_is_root_` (root functions), only if value is Array (add/delete only, skip property changes), create DiffFunction with name and returnType from `value[0]`
   - Sort entities and functions by name (localeCompare) before returning
   - If delta is undefined/null, return `{ entities: [], functions: [] }` (empty diff)
  </action>

  <action>
1. Create `app/src/lib/diff/fetch-historical.ts`:
   - Export `fetchHistoricalBlob(year: number, month: number): Promise<Metadata | null>`
   - Import `decompressFromUTF16` from `lz-string`, `getHistoricalBlobUrl` from `@/lib/constants`, `Metadata` from `@/lib/metadata/types`
   - Implementation follows the exact same pattern as `fetchFresh()` in `boot.ts`:
     ```
     const url = getHistoricalBlobUrl(year, month)
     const res = await fetch(url)
     if (res.status === 404) return null  // blob doesn't exist for this month
     if (!res.ok) throw new Error(`HTTP ${res.status}: ${res.statusText}`)
     const compressed = await res.text()
     const json = decompressFromUTF16(compressed)
     if (json === null) throw new Error('Decompression failed')
     return JSON.parse(json) as Metadata
     ```
   - **Key difference from boot.ts:** Returns `null` on 404 (not throw) — per CONTEXT.md decision for graceful handling of missing historical blobs
   - **Throws** on non-404 network errors (diff-store will catch and set error status)

2. Export a helper `getDefaultComparisonDate(): { year: number; month: number }` that returns the year/month for "one month ago" from the current date. E.g., in Feb 2026: `{ year: 2026, month: 1 }`. Handle January wrapping (Jan 2026 → Dec 2025). This will be the default comparison target.
