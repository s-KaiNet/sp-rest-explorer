**EntityChangeCard.tsx — Entity name linking:**

Import `Link` from `react-router`.

In the card header, make the entity name conditionally clickable:
- If `entity.changeType` is `'added'` or `'updated'`: Wrap the name in a `<Link to={"/entity/" + encodeURIComponent(entity.name)}>` component. Style: `font-mono text-sm font-bold hover:underline cursor-pointer` (keep existing font-mono text-sm font-bold, add hover:underline). Do NOT change the text color — per CONTEXT.md "No always-visible link color — keeps cards clean". The link should be subtle: only underline on hover.
- If `entity.changeType` is `'removed'`: Keep as plain `<span>` (existing behavior). Removed entities no longer exist in current metadata, so linking to them would show a 404/not-found in the types view.

The entity name element currently is:
```tsx
<span className="min-w-0 truncate font-mono text-sm font-bold">{entity.name}</span>
```

Replace with a conditional:
```tsx
{entity.changeType !== 'removed' ? (
  <Link
    to={`/entity/${encodeURIComponent(entity.name)}`}
    className="min-w-0 truncate font-mono text-sm font-bold hover:underline"
    onClick={(e) => e.stopPropagation()}
    title={`View ${entity.name} details`}
  >
    {entity.name}
  </Link>
) : (
  <span className="min-w-0 truncate font-mono text-sm font-bold text-muted-foreground">
    {entity.name}
  </span>
)}
```

Note the `onClick={(e) => e.stopPropagation()}` — this prevents the card expand/collapse toggle from firing when clicking the entity name link. Without this, clicking the link would both navigate AND toggle the card.

Also add `text-muted-foreground` to removed entity names to visually indicate they're not actionable.

**RootFunctionsTable.tsx — Function name linking:**

Import `Link` from `react-router`.

In each table row, make the function name conditionally clickable:
- If `fn.changeType` is `'added'` or `'updated'`: Wrap the CodeText in a `<Link to="/_api/">` component. All root functions link to the Explore API root page (not individual function URLs — functions don't have dedicated routes, they appear on the /_api/ page).
- If `fn.changeType` is `'removed'`: Keep as plain text. Removed functions no longer exist.

The function name cell currently is:
```tsx
<td className="truncate border-b border-border/50 py-1.5 pr-3">
  <CodeText variant="fn">{fn.name}</CodeText>
</td>
```

Replace with:
```tsx
<td className="truncate border-b border-border/50 py-1.5 pr-3">
  {fn.changeType !== 'removed' ? (
    <Link to="/_api/" className="hover:underline" title={`View in Explore API`}>
      <CodeText variant="fn">{fn.name}</CodeText>
    </Link>
  ) : (
    <span className="text-muted-foreground">
      <CodeText variant="fn">{fn.name}</CodeText>
    </span>
  )}
</td>
```

Style the removed function names with `text-muted-foreground` wrapper to visually indicate they're inactive.

- Cursor pointer (inherent from `<Link>`)
- No always-visible link color — text stays the same color as before
- Removed items get `text-muted-foreground` to appear dimmed
