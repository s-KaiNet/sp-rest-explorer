  <name>Task 1: Implement complete ChangelogPage with all states</name>
  <files>app/src/pages/ChangelogPage.tsx</files>
  <action>
Replace the entire placeholder ChangelogPage.tsx with a complete implementation. The page has these concerns:

**Lifecycle:**
- On mount: call `computeDiff(year, month)` using `getDefaultComparisonDate()` for the default comparison period. Use `useEffect` with cleanup that calls `resetDiff()`.
- Read `useParams()` for optional `monthKey` param (format: `2026-1`). If present, parse year/month from it; otherwise use `getDefaultComparisonDate()`.

**Layout (match HowItWorksPage pattern):**
- Outer wrapper: `<div className="flex-1 overflow-y-auto">`
- Inner container: `<div className="mx-auto max-w-[720px] px-6 py-10">`
- Page heading: "API Changelog" as `<h1>` with same styling as HowItWorksPage (`text-[28px] font-bold text-foreground mb-2`)
- Subtitle: "Changes in {Month Year}" (e.g., "Changes in January 2026") using the comparison date — `text-[15px] text-muted-foreground`

**State rendering (driven by `useDiffStatus()`):**

1. **Loading state** (`status === 'loading'` or `status === 'idle'`):
   - Centered spinner using the same pattern as ContentSkeleton.tsx (`<div className="size-8 rounded-full border-3 border-muted border-t-muted-foreground animate-spin" />`)
   - Below spinner: "Computing changes..." text in `text-sm text-muted-foreground`
   - Wrap in a flex container: `flex flex-col items-center justify-center py-20 gap-3`

2. **Ready state with data** (`status === 'ready'` and diff has changes):
   - **Summary bar:** 3 stat cards in a horizontal row using `flex gap-4`
   - Each card: `flex-1 rounded-xl border border-border bg-background p-5 text-center`
   - Card content: large count number (`text-3xl font-bold leading-none mb-1`) + label text (`text-xs font-medium text-muted-foreground`)
   - Color scheme via text color on the count number:
     - Added card: `text-green-600 dark:text-green-400`, label "Added"
     - Updated card: `text-blue-600 dark:text-blue-400`, label "Updated"  
     - Removed card: `text-red-600 dark:text-red-400`, label "Removed"
   - Count calculation: for each change type, sum entities + functions. E.g., Added count = `diff.entities.filter(e => e.changeType === 'added').length + diff.functions.filter(f => f.changeType === 'added').length`
   - Always show all 3 cards, even when count is 0 (per CONTEXT.md decision: "no layout shifting, communicates loaded correctly")
   - Below summary bar: a placeholder message for detail views (Phase 26): `<p className="mt-8 text-center text-sm text-muted-foreground">Detailed change views coming in a future update.</p>`

3. **Ready state with empty data** (`status === 'ready'` and all counts are zero):
   - Still show the summary bar with all zeros (per CONTEXT.md decision)
   - Below summary bar: empty state visual
   - Muted icon centered: use `ClipboardCheck` from lucide-react, `className="size-12 text-muted-foreground/40"`
   - Heading: "No changes detected" — `text-lg font-medium text-foreground mt-3`
   - Subtext: "No API changes were found for {Month Year}. The diff is computed by comparing monthly metadata snapshots." — `text-sm text-muted-foreground mt-1 max-w-md text-center`
   - Wrap in: `flex flex-col items-center py-12`

4. **Error state** — differentiate by error message content:
   - **404 / missing blob** (when `diffError` contains "404" or when diff result is empty due to 404 — note: per Phase 24 design, 404 actually produces empty DiffChanges, not error. So this error path handles non-404 fetch failures):
     Actually, re-reading Phase 24: 404 → `setReady({ entities: [], functions: [] })`, NOT setError. So the error state ONLY fires on actual network errors.
   - **Network error** (`status === 'error'`):
     - Icon: `AlertCircle` from lucide-react, `className="size-12 text-destructive"`
     - Heading: "Unable to load changelog data" — `text-lg font-medium text-foreground mt-3`
     - Subtext: Display the `diffError` message or fallback "An error occurred while computing changes." — `text-sm text-muted-foreground mt-1 max-w-md text-center`
     - "Try again" button: calls `computeDiff(year, month)` again. Style: `inline-flex items-center gap-2 rounded-md bg-primary px-4 py-2 text-sm font-medium text-primary-foreground hover:bg-primary/90 transition-colors mt-4`
     - Use `RefreshCw` icon in the button, matching ErrorState.tsx pattern
   - Wrap error in: `flex flex-col items-center py-20`

**Helper function:** Create a `getMonthLabel(year: number, month: number): string` helper that returns "January 2026" format using `new Date(year, month - 1).toLocaleString('en-US', { month: 'long', year: 'numeric' })`.

**Imports needed:**
- `{ useEffect, useMemo }` from 'react'
- `{ useParams }` from 'react-router'
- `{ AlertCircle, ClipboardCheck, RefreshCw }` from 'lucide-react'
- `{ useDiffSnapshot, useDiffStatus, useDiffError, computeDiff, resetDiff, getDefaultComparisonDate }` from '@/lib/diff'

**What NOT to do:**
- Do NOT add fade-in animations (per CONTEXT.md: "instant swap from loading to content")
- Do NOT add skeleton cards for loading (per CONTEXT.md: "centered spinner with message text, not skeleton cards")
- Do NOT suggest actions in the empty state (per CONTEXT.md: "No action suggested — purely informational; range selection comes in Phase 27")
- Do NOT modify routes.tsx or Header.tsx (route and nav link already exist)
- Do NOT manage data fetching state locally — use the diff-store reactive hooks exclusively
