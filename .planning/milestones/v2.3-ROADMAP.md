## Phases

### Phase 29: Automated GitHub Pages Deployment

**Goal**: Frontend deploys automatically to GitHub Pages on push — no more committed build output in the repository
**Depends on**: Nothing (first phase of v2.3)
**Plans**: 2 plans

Plans:

- [x] 29-01: CI/CD pipeline + build config + .gitignore (Wave 1) — completed 2026-02-25
- [x] 29-02: Delete committed docs/ folder (Wave 2) — completed 2026-02-25

**Details:**

Phase 29 delivered a GitHub Actions workflow (`.github/workflows/deploy.yml`) that builds and deploys the frontend on every push to master. The Vite build output was redirected from `../docs` to `app/dist/` (inside the project root). The `.gitignore` was updated to block both `docs/` and `app/dist/`. CI strict mode uncovered two TypeScript issues (objectHash return type, ES2023 lib for toSorted) that were fixed. The committed `docs/` directory (5 files) was removed in Plan 02. Live site verified working at https://nicklasmansson.github.io/sp-rest-explorer/.
