<task type="auto">
  <name>Task 2: Subdue dark mode borders on Cmd+K search modal</name>
  <files>app/src/index.css, app/src/components/search/CommandPalette.tsx</files>
  <action>
  Per user decisions:
  - Modal-only fix — override border color specifically on the CommandPalette/dialog, no global `--border` change
  - Subdue all modal borders: outer dialog border, input separator (`border-b`), footer separator (`border-t`), and `<kbd>` hint borders
  - Target: subtly visible borders — still present but noticeably dimmer than current
  - Backdrop/overlay stays unchanged — only borders are adjusted

  **Approach:** Add a CSS custom property for subdued modal borders in dark mode, then apply it via Tailwind classes on the CommandPalette elements.

  **Step 1 — Add CSS variable** in `app/src/index.css`, inside the `.dark` block:
  ```css
  /* Subdued modal borders — dimmer than standard --border for less visual noise */
  --modal-border: oklch(0.24 0.01 260);
  ```
  This is darker than the standard `--border` (`oklch(0.30 0.01 260)`) but still visible.

  Also add to the `@theme inline` block:
  ```css
  --color-modal-border: var(--modal-border);
  ```

  And in the `:root` (light mode), add a fallback that equals normal border:
  ```css
  --modal-border: oklch(0.922 0 0);
  ```

  **Step 2 — Apply to CommandPalette.tsx** borders:

  a) **Outer dialog border**: In the `CommandDialog` call, add `dark:border-modal-border` to the className:
     ```tsx
     className="top-[10%] translate-y-0 sm:max-w-2xl dark:border-modal-border"
     ```
     Wait — the `border` class is on `DialogContent` inside `command.tsx`, not on the `CommandDialog` prop. Since we don't want to modify the shared UI component, instead pass the border override via the `className` prop which gets forwarded to `DialogContent`:
     ```tsx
     className="top-[10%] translate-y-0 sm:max-w-2xl [&]:dark:border-modal-border"
     ```

  b) **Input separator** (`border-b` on the command-input-wrapper div): Add a dark mode override class in CommandPalette.tsx on the `CommandInput` or via the `CommandDialog`'s className cascading selector. The cleanest approach: add to the CommandDialog className a descendant selector:
     ```
     dark:[&_[data-slot=command-input-wrapper]]:border-modal-border
     ```

  c) **Footer separator** (the `border-t border-border` div at the bottom of CommandPalette.tsx, line 466): Change to:
     ```tsx
     className="flex items-center gap-3 border-t border-border dark:border-modal-border px-3 py-2 text-xs text-muted-foreground"
     ```

  d) **`<kbd>` hint borders** (the ESC kbd in CommandInput suffix and the arrow/enter/esc kbds in footer): These use `border border-border`. Add `dark:border-modal-border` to each:
     - ESC kbd (line 419): add `dark:border-modal-border`
     - Footer kbds (lines 468, 474, 480): add `dark:border-modal-border`

  **Summary of changes to CommandPalette.tsx:**
  - Line ~410 (CommandDialog className): add `dark:[&_[data-slot=command-input-wrapper]]:border-modal-border`
  - Line ~419 (ESC kbd): add `dark:border-modal-border`
  - Line ~466 (footer div): add `dark:border-modal-border`
  - Lines ~468, 474, 480 (footer kbds): add `dark:border-modal-border`

  This scopes ALL border dimming to the command palette only. No other modals or UI elements affected.
  </action>
