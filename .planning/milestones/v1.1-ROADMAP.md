# Milestone v1.1: Search, Types & Polish

**Status:** SHIPPED 2026-02-15
**Phases:** 6-8 (including 07.1, 07.2 inserted phases)
**Total Plans:** 13

## Overview

Added global deep search via Cmd+K command palette, a full Explore Types browsing surface with namespace-grouped sidebar and type detail views, and quality-of-life polish (How It Works page, copy-to-clipboard, GitHub Dark dark mode, favicons). Extended the v1.0 MVP into a complete SharePoint REST API exploration tool.

## Phases

### Phase 6: Global Search

**Goal**: Users can find any entity, function, or property across all 6K+ indexed items in seconds and jump directly to it.
**Depends on**: None (MiniSearch index with ~6K items already exists in data layer)
**Plans**: 2 plans

Plans:
- [x] 06-01: Install shadcn primitives and build CommandPalette component
- [x] 06-02: Wire triggers (Cmd+K, header, hero) and integrate with app

**Success Criteria:**
1. User presses Cmd+K (Mac) or Ctrl+K (Windows) from any page and a command palette opens immediately
2. User types a query and sees matching results appear in real-time as they type, with no perceptible delay
3. User sees results organized into groups by kind (entities, functions, properties) so they can scan to the right category
4. User selects a result (click or keyboard) and the app navigates to that item's detail view with correct context (sidebar, breadcrumbs)
5. User presses Escape or clicks outside to dismiss the palette and return to their previous view unchanged

**Requirements:** SRCH-01, SRCH-02, SRCH-03, SRCH-04

### Phase 7: Explore Types

**Goal**: Users can browse, inspect, and cross-reference every complex type in the SharePoint REST API metadata — with namespace-grouped sidebar, type detail views, inheritance display, precomputed used-by index, and cross-navigation via TypeLink.
**Depends on**: Phase 6 not required, but TypeLink component from v1.0 will be extended
**Plans**: 3 plans

Plans:
- [x] 07-01: Data layer (type indexes) + ComplexTypeDetail component
- [x] 07-02: TypesPage sidebar with namespace grouping + TypeLink cross-navigation
- [x] 07-03: Gap closure: sidebar lists all types, fullName filter, entity derived types

**Success Criteria:**
1. User clicks "Explore Types" in the main header navigation and sees a sidebar listing all complex types grouped by namespace, with a filter input to narrow the list
2. User clicks a complex type and sees its properties table with Name, Type, Nullable columns
3. User viewing a type can see its base type as a clickable link and any derived types as a list
4. User viewing a type can see a "Used by" section showing which entities reference this type via navigation properties (precomputed, not scanned)
5. User clicks any type reference in the Explore API views and navigates to that type's detail in Explore Types with sidebar synced

**Requirements:** TYPE-01, TYPE-02, TYPE-03, TYPE-04, TYPE-05, TYPE-06

## Milestone Summary

**Decimal Phases:**
- Phase 07.1: Fix search experience (inserted after Phase 7 for urgent search navigation UX fix)
- Phase 07.2: Add path to API Endpoints index (inserted after Phase 07.1 for path-based search)

**Key Decisions:**
- 3 core phases for 13 requirements (search, types, polish — natural clustering)
- Search first (Phase 6) — highest user value, data layer already complete
- Types second (Phase 7) — largest feature surface, extends browsing capability
- Polish last (Phase 8) — independent small items, completes milestone
- shouldFilter=false on cmdk — MiniSearch handles all filtering
- BFS tree-walk for comprehensive endpoint indexing (~62K reachable, ~3,330 unique)
- Substring matching over MiniSearch tokenization for path search (07.2-02 gap closure)
- Type classification heuristic: complex types = no nav props AND no function IDs AND not Collection wrappers
- GitHub Dark palette as dark mode reference with blue OKLCH undertones
- Hardcoded stats in How It Works page (not computed from live metadata)

**Issues Resolved:**
- Search result navigation UX — pre-computed paths replaced runtime path resolution
- UsedByBar O(n*m) scan replaced with precomputed O(1) index
- MiniSearch tokenization issues for path search replaced with substring filtering
- UAT gaps: diagram overflow, missing header logo, dark mode chrome elevation

**Issues Deferred:**
- SRCH-05: Recently visited items in command palette (deferred to future milestone)
- CHLG-01 through CHLG-06: API Changelog view (deferred to future milestone)
- ADDL-02: GitHub Actions CI/CD auto-deployment (deferred to future milestone)

**Technical Debt Incurred:**
- TypeLink navigates to /entity/{fullName} for all types — no entity-to-API-path resolver built (pragmatic tradeoff)
- Recently visited kind mapping relies on depth heuristic (depth 2 = root)
