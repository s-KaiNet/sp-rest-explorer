## Overview

Rewrote the Azure Functions backend from scratch — modern TypeScript Azure Functions v4 project with MSAL certificate-based client credentials auth, resilient metadata fetch with retry/backoff/timeout, XML-to-JSON parsing pipeline (byte-identical to legacy), lz-string compression, simplified blob layout (single `api-files` container), and daily timer with HTTP trigger. Deployed to Azure with Key Vault-backed certificates and managed identity.

## Milestone Summary

**Decimal Phases:** None — phases 18-21 executed sequentially as planned.

**Key Decisions:**
- Decision: Used `backend/` directory name (Rationale: Clearer separation from frontend `app/`)
- Decision: MSAL thumbprintSha256 over deprecated thumbprint (Rationale: Modern API, future-proof)
- Decision: Single interfaces.ts file for all 7 metadata types (Rationale: Reduces import complexity)
- Decision: Plain async function for parser over class (Rationale: Cleaner API, no stateful coupling)
- Decision: Vitest as test runner (Rationale: TypeScript-native, zero config)
- Decision: Shared handler pattern for timer+HTTP triggers (Rationale: Single orchestration source)
- Decision: Key Vault for PEM certificates (Rationale: Azure app settings mangle multiline values)
- Decision: System-assigned managed identity (Rationale: No secrets to rotate or leak)
- Decision: Buffer.byteLength for upload content length (Rationale: Critical for multi-byte lz-string output)

**Issues Resolved:**
- func tools v4.7.0 NormalizeDotnetFrameworkVersion bug (workaround: --dotnet-version 8.0)
- package.json main field was unresolvable glob pattern (fixed to dist/src/index.js)
- Empty src/index.ts prevented function discovery (added explicit imports)
- .funcignore excluded node_modules runtime deps (removed node_modules from ignore)
- PEM certificates can't be stored as plain Azure app settings (Key Vault solution)

**Issues Deferred:**
- Frontend compression switch to .zip.json (FRNT-01) — separate milestone
- GitHub Actions CI/CD (ADDL-02) — separate milestone
- API Changelog (CHLG-01 through CHLG-06) — separate milestone
