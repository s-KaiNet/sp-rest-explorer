<task type="auto">
  <name>Task 2: Create auth module and HTTP trigger function</name>
  <files>
    backend/src/auth.ts
    backend/src/functions/validateAuth.ts
  </files>
  <action>
    **src/auth.ts — Auth module (reused by Phase 19+):**

    Create a `getToken()` function that acquires a SharePoint access token using MSAL certificate-based client credentials:

    1. Helper `getRequiredEnv(name: string): string` — reads process.env, throws descriptive error if missing
    2. Helper `computeThumbprint(certPem: string): string` — uses `X509Certificate` from `node:crypto` to compute SHA-256 thumbprint. The `.fingerprint256` property returns colon-separated hex (e.g., "AB:CD:EF:...") — strip colons and lowercase for MSAL's `thumbprintSha256` field.
    3. Export `async function getToken(): Promise<string>`:
       - Read env vars: SP_CERT_PEM, SP_KEY_PEM, ENTRA_CLIENT_ID, ENTRA_TENANT_ID, SP_URL
       - Handle PEM newline corruption: replace literal `\n` with actual newlines (`pem.replace(/\\n/g, '\n')`), strip carriage returns (`pem.replace(/\r/g, '')`)
       - Compute thumbprint from cert PEM
       - Create MSAL `ConfidentialClientApplication` with config: clientId, authority (`https://login.microsoftonline.com/${tenantId}`), clientCertificate with `thumbprintSha256` (NOT deprecated `thumbprint`) and `privateKey`
       - Call `acquireTokenByClientCredential` with scopes: [`${spUrl}/.default`] — the `.default` scope is REQUIRED for client credentials flow against SharePoint
       - Throw descriptive error if result or accessToken is null
       - Return the access token string

    **src/functions/validateAuth.ts — HTTP trigger:**

    1. Import `dotenv/config` at the very top (before any other imports) — loads .env file for local dev
    2. Import `app, HttpRequest, HttpResponseInit, InvocationContext` from `@azure/functions`
    3. Import `axios` for HTTP requests
    4. Import `getToken` from `../auth`
    5. Create async handler function `validateAuth`:
       - Log "Auth validation triggered" via `context.log`
       - Call `getToken()` to acquire access token
       - Make GET request to `${process.env.SP_URL}/_api/web` with headers: `Authorization: Bearer ${token}`, `Accept: application/json;odata=verbose`
       - Return `{ status: 200, jsonBody: response.data }` on success
       - Catch errors: extract message from Error instance, log via `context.log.error`, return `{ status: 500, jsonBody: { error: message } }`
    6. Register with `app.http('validateAuth', { methods: ['GET'], authLevel: 'function', handler: validateAuth })`
       - authLevel MUST be 'function' (not 'anonymous') per user decision

    After creating both files, run `npm run build` to verify compilation.
  </action>
