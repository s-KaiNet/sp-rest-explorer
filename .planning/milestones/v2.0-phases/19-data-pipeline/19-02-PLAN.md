  <action>
**Create `backend/src/pipeline/parse-metadata.ts`** exporting `async function parseMetadata(xml: string): Promise<Metadata>`.

Port the legacy `MetadataParser` class from `az-funcs/src/metadataParser.ts` (302 lines) method-by-method. The function should NOT be a class — use a plain async function with inner helper functions (Claude's discretion on organization). However, if a class helps organize the internal Maps, that's fine too.

**CRITICAL BEHAVIORS to replicate exactly (from RESEARCH.md):**

1. **Alias map** — exact same 6 entries as legacy lines 14-21. Applied to function names before underscore replacement.

2. **Function ID assignment** — `let i = 1`, increment via `i++` in the FunctionImport object literal. The ID is consumed BEFORE the Internal filter check. Internal functions consume IDs but are excluded from output. This creates gaps in the ID sequence.

3. **Internal filtering** — Skip any function where `name.indexOf('Internal') !== -1` OR `returnType.indexOf('Internal') !== -1`. Applied AFTER ID assignment but BEFORE alias check.

4. **Alias dedup two-branch logic:**
   - If `aliases[name]` exists AND no function already has that alias as its name → rename function to alias
   - If `aliases[name]` exists AND a function already HAS that alias as its name → skip this function entirely (`continue`)

5. **underscore-to-dot replacement** — `name.replace(/_/gi, '.')` applied AFTER alias resolution. Aliased names like `social.following` don't have underscores so are unaffected.

6. **`isRoot` deletion** — Create with `isRoot: true` default. Set `isRoot = false` if `this` parameter found. Then `delete funcImport.isRoot` when `isRoot` is `false`. This means: root functions have `isRoot: true` in JSON, non-root functions have NO `isRoot` key.

7. **`isBindable` and `isComposable`** — Set from XML attributes: `func.$.IsBindable === 'true'` when attribute exists, `undefined` when it doesn't. `undefined` values are omitted from JSON serialization.

8. **`nullable` on Parameter** — Uses the FUNCTION-level `func.$.Nullable`, NOT the parameter-level attribute. This applies the same nullable to ALL parameters of a function. Looks like a legacy bug but MUST be replicated: `nullable: func.$.Nullable ? func.$.Nullable === 'true' : undefined`

9. **Association extraction** — For each schema with Association elements: extract End[0] and End[1] roles. If multiplicity contains `*`, wrap type in `Collection()`. Store in Map by association name.

10. **Entity/ComplexType extraction** — `createEntityTypeWithProperties()` handles both. Uses `${namespace}.${type.$.Name}` as fullName. Stores in Map. Extracts Property array. Sets `baseTypeName` from `$.BaseType` if present.

11. **NavigationProperty resolution** — Uses Association lookup. Extracts relationship name by `substring(lastIndexOf('.') + 1)`. Looks up association, gets `roles[toRole]` for the target type.

12. **Synthetic Collection() entities** — `ensureCollectionEntityType()` creates minimal entity for any `Collection(...)` type referenced by properties, navProperties, or function returnTypes that doesn't already exist as an entity. Minimal entity: `{ fullName: typeName, name: typeName, properties: [], functionIds: [], navigationProperties: [] }`.

13. **Function-to-entity linking** — `populateEntitiesWithMethods()` — for each function, find `this` parameter. If found, `ensureCollectionEntityType` for that type, then push `func.id` to `entity.functionIds`.

14. **Extraction order matters:** associations → types (Complex + Entity) → function imports → populate collection objects → populate entities with methods → convert Maps to plain objects.

15. **Map-to-object conversion:** `entities.forEach((entity, key) => { metadata.entities[key] = entity })` and `functions.forEach((func) => { metadata.functions[func.id] = func })`.

16. **Use `parseStringPromise` from xml2js** — replaces Bluebird promisify. Access parsed XML via `obj['edmx:Edmx']['edmx:DataServices'][0]['Schema']`.

17. **JSON compact format** — `JSON.stringify(metadata)` with no indentation args. This is the caller's responsibility but parseMetadata returns the Metadata object directly.

18. **Drop dead code** — Do NOT port the commented-out sorting TODO (lines 52-63) or the commented-out underscore `continue` (line 171-173). Clean rewrite per CONTEXT.md.

Import all interfaces from `./interfaces.js`. Use `.js` extensions in all relative imports per Node16 module resolution.
