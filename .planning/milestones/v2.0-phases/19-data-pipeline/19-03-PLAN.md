**Create `backend/src/pipeline/compress.ts`:**

Export `function compressJson(json: string): string` that:
1. Calls `compressToUTF16(json)` from lz-string
2. Returns the compressed string

Import `{ compressToUTF16 }` from `'lz-string'`.

This is intentionally a thin wrapper — it gives a named function for the pipeline and a single place to change compression strategy if needed.

**Create `backend/src/pipeline/index.ts`:**

This is the pipeline orchestrator and barrel export. It must:

1. **Re-export everything** for Phase 20 consumption:
   ```typescript
   export { fetchMetadataXml } from './fetch-metadata.js';
   export { parseMetadata } from './parse-metadata.js';
   export { compressJson } from './compress.js';
   export type { Metadata, EntityType, FunctionImport, Property, NavigationProperty, Parameter, Association } from './interfaces.js';
   ```

2. **Define `PipelineResult` interface:**
   ```typescript
   export interface PipelineResult {
     readonly xml: string;          // Raw XML from SharePoint
     readonly json: string;         // Compact JSON.stringify(metadata)
     readonly compressedJson: string; // lz-string compressToUTF16(json)
   }
   ```

3. **Export `runPipeline` function:**
   ```typescript
   export async function runPipeline(token: string, spUrl: string): Promise<PipelineResult> {
     const xml = await fetchMetadataXml(token, spUrl);
     const metadata = await parseMetadata(xml);
     const json = JSON.stringify(metadata);  // Compact, no indentation (PROC-03)
     const compressedJson = compressJson(json);
     return { xml, json, compressedJson };
   }
   ```

Use `.js` extensions in all relative imports per Node16 module resolution.
  </action>

Run `npx tsc --noEmit` in `backend/` — zero errors. Verify all exports are accessible.
  </verify>
