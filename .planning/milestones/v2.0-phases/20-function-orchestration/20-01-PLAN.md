<task type="auto">
  <name>Task 1: Install @azure/storage-blob and create blob upload module</name>
  <files>backend/package.json, backend/src/upload/blob-uploader.ts</files>
  <action>
1. Install the Azure Blob Storage SDK:
   ```bash
   cd backend && npm install @azure/storage-blob
   ```

2. Create `backend/src/upload/blob-uploader.ts` with:

   a. `BlobDefinition` interface: `{ name: string; content: string; contentType: string }`
      Export this interface for test consumption.

   b. `buildBlobList(result: PipelineResult, now: Date): BlobDefinition[]` — EXPORTED pure function:
      - Compute `year = now.getUTCFullYear()`, `month = now.getUTCMonth() + 1` (1-indexed per user decision)
      - Build prefix: `${year}y_m${month}_metadata`
      - Return array of exactly 6 blob definitions in this order:
        1. `metadata.latest.json` → `result.json` → `application/json`
        2. `metadata.latest.xml` → `result.xml` → `application/xml`
        3. `metadata.latest.zip.json` → `result.compressedJson` → `application/json`
        4. `${prefix}.json` → `result.json` → `application/json`
        5. `${prefix}.xml` → `result.xml` → `application/xml`
        6. `${prefix}.zip.json` → `result.compressedJson` → `application/json`
      - Content-Type per user decision: `application/json` for .json and .zip.json, `application/xml` for .xml (BLOB-08)

   c. `uploadBlobs(context: InvocationContext, result: PipelineResult, now: Date): Promise<void>` — EXPORTED:
      - Read connection string from `process.env.AzureWebJobsStorage` — throw descriptive error if missing
      - Create `BlobServiceClient.fromConnectionString(connectionString)`
      - Get container client for `'api-files'` (CONTAINER_NAME constant)
      - Call `containerClient.createIfNotExists({ access: 'blob' })` — idempotent, self-healing (BLOB-07)
      - Call `buildBlobList(result, now)` to get the 6 blob definitions
      - Loop sequentially (for loop, NOT Promise.all) over blobs:
        - Get `BlockBlobClient` via `containerClient.getBlockBlobClient(blob.name)`
        - Compute content length: `Buffer.byteLength(blob.content, 'utf-8')` — CRITICAL: not `string.length` (research pitfall: lz-string compressToUTF16 output has multi-byte chars)
        - Call `blockBlobClient.upload(blob.content, contentLength, { blobHTTPHeaders: { blobContentType: blob.contentType } })`
        - Log: `context.log(\`[upload] ${blob.name} uploaded (${i + 1}/${blobs.length})\`)`
      - If any upload fails, let the error propagate (stop uploading remaining, no rollback of already-uploaded per user decision)

   d. Imports needed:
      - `{ BlobServiceClient } from '@azure/storage-blob'`
      - `{ InvocationContext } from '@azure/functions'`
      - `{ PipelineResult } from '../pipeline/index.js'` (note `.js` extension for Node16 module resolution)
  </action>
