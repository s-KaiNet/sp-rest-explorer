# Milestone v1.2: UI Improvements

**Status:** ✅ SHIPPED 2026-02-15
**Phases:** 9-10
**Total Plans:** 5

## Overview

Polish the UI across Explore API, Explore Types, home page, and command palette to fix visual bugs, improve navigation consistency, and add missing UX elements. Removed clutter badges, added namespace grouping to Explore API sidebar, branded the home page, redesigned the Explore API welcome screen, and subdued dark mode modal borders.

## Phases

### Phase 9: Explore API Sidebar Polish ✅ (2026-02-15)

**Goal**: The Explore API sidebar presents root-level items cleanly — grouped by namespace, with consistent badge positioning, no visual clutter from type badges, and no animation glitches.
**Depends on**: None (sidebar components exist from v1.0)

**Requirements:** SIDE-01, SIDE-02, SIDE-03, VISU-02

**Success Criteria:**
1. User navigates forward in the Explore API sidebar and sees smooth slide animation with no horizontal scrollbar flickering
2. User viewing the Explore API root level sees items organized into collapsible namespace groups (matching Explore Types pattern), with ungrouped items shown first under "No Group"
3. User sees root indicator badges on the right side of sidebar items, aligned consistently with function/nav property badges
4. User no longer sees "Entity Type" or "Complex Type" badges anywhere in the app — sidebar items, detail views, or search results

Plans:
- [x] 09-01: Remove Entity Type, Complex Type, and Composable badges from all locations
- [x] 09-02: Namespace grouping, badge repositioning, and animation fix for sidebar
- [x] 09-03: Fix namespace grouping to use entry.name instead of entry.returnType (gap closure)

**Details:**
- 09-01: Removed "Entity Type" badge from EntityDetail.tsx, "Complex Type" badge from ComplexTypeDetail.tsx, and "COMPOSABLE" badges from MethodsTable.tsx and ExplorePage.tsx inline function view. 4 files modified.
- 09-02: Added namespace-grouped collapsible sidebar for root level (~200+ endpoints organized into Core first, then alphabetical groups). Root indicator badge repositioned to right side. Horizontal scrollbar flickering fixed via overflow-x-hidden. Scroll-to-top reset on navigation. 3 files modified.
- 09-03: Fixed getNamespace() to extract from entry.name (item's fully-qualified identifier) instead of entry.returnType. Corrected 74% of root entries (590/793) that were grouped under wrong namespaces. 1 file modified.

## Milestone Summary

**Key Decisions:**
- Badges removed entirely with no replacement — item names and context are sufficient
- Namespace extraction uses item's own fully-qualified name (entry.name), not return type
- Replicated TypesSidebar visual pattern in Sidebar.tsx rather than sharing component (different data types)
- Hardcoded approximate stats on home page for instant render; live computed stats on Explore API page (data already loaded)
- Scoped --modal-border CSS variable for per-modal dark mode border control (no global --border change)

**Issues Resolved:**
- Horizontal scrollbar flickering during sidebar slide animation (overflow-x-hidden)
- 74% of root entries (590/793) misclassified into wrong namespace groups (entry.name fix)
- Bright dark mode borders on Cmd+K search modal (scoped CSS variable)

**Issues Deferred:**
- CHLG-01 through CHLG-06: API Changelog view (deferred to future milestone)
- ADDL-02: GitHub Actions CI/CD auto-deployment (deferred to future milestone)

**Technical Debt Incurred:**
- None — v1.2 was a polish milestone that resolved debt rather than creating it
