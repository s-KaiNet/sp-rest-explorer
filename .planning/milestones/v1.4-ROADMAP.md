# Milestone v1.4: Unify Icons

**Status:** ✅ SHIPPED 2026-02-19
**Phases:** 13-17
**Total Plans:** 7

## Overview

Replaced all text symbols and special characters with consistent Lucide icons across the entire app. Introduced a TypeIcon component with dedicated CSS color tokens for all 4 API types (root=green, nav property=purple, function=blue, type/entity=orange/amber). Updated every view — Explore API sidebar, search modal, home page, Explore Types — for complete visual consistency. Changed entity link colors from hardcoded emerald to the type-entity CSS variable. Reorganized search modal footer layout.

## Phases

### Phase 13: Icon System Foundation ✅ (2026-02-18)

**Goal**: A reusable icon system exists for all 4 API types — each type has a distinct Lucide icon and CSS color token, ready to be used everywhere
**Depends on**: Nothing (foundational work for v1.4)

**Requirements:** ICON-01, ICON-02, ICON-03, ICON-04

**Success Criteria:**
1. A `TypeIcon` component renders a distinct Lucide icon for each of the 4 API types: root, nav property, function, and type/entity
2. Each icon renders in its designated type color (root = green, nav property = purple, function = blue, type/entity = orange/amber)
3. A `--type-root` CSS custom property exists and produces a green color (OKLCH hue ~155) distinct from the other type colors
4. The `--type-entity` CSS custom property produces an orange/amber color (OKLCH hue ~75-85) clearly distinguishable from the root green

Plans:
- [x] 13-01: CSS color tokens + ApiType type + TypeIcon component
- [x] 13-02: Root-type color fix — migrate all root/entity CSS classes to correct type tokens

**Details:**
- 13-01: Created 4 CSS custom properties with OKLCH values and dark mode variants. ApiType union type in api-types.ts. TypeIcon component with Record-based icon/color lookup maps, 3 size variants (sm/md/lg), unknown type fallback. 3 files.
- 13-02: 7 CSS class replacements across 4 files so all root-type UI elements render green (type-root) and entity elements render orange (type-entity). Fixed root recently-visited cards, browse-all button, sidebar badge, welcome screen, and search palette root pill.

## Milestone Summary

**Key Decisions:**
- ApiType kept as simple union type — no enum, no utility functions
- TypeIcon uses Record maps for icon/color lookup instead of if/else chains
- OKLCH chroma values 0.12-0.15 for muted/pastel appearance of new colors
- Caller passes apiType prop to SidebarItem — no internal kind-to-type mapping
- Welcome hero uses bare TypeIcon with no background container
- Root items in search results use green Box icon as sole indicator — no pill badge
- Recently visited cards use TypeIcon md (20px) — proportional to card size
- Hint box text color corrected from green (root) to amber (entity) to match semantics
- Entity link color changed from hardcoded emerald to --type-entity CSS variable
- Footer hint bar split with justify-between instead of spacer element

**Issues Resolved:**
- Root UI elements rendering in wrong color (orange instead of green) — fixed with dedicated --type-root token
- Explore Types hint box using green (root) color instead of amber (entity) — corrected
- Inconsistent text symbols across views (<>, ƒ, NAV, T, Root pill) — all replaced with Lucide icons

**Issues Deferred:**
- CHLG-01 through CHLG-06: API Changelog view (deferred to future milestone)
- ADDL-02: GitHub Actions CI/CD auto-deployment (deferred to future milestone)

**Technical Debt Incurred:**
- TypeLink navigates to /entity/{fullName} for all types — no entity-to-API-path resolver (pre-existing)
- Recently visited kind mapping relies on depth heuristic (depth 2 = root) (pre-existing)
