<task type="auto">
  <name>Task 1: Add copy-to-clipboard button to BreadcrumbBar</name>
  <files>app/src/components/navigation/BreadcrumbBar.tsx</files>
  <action>
Add a copy button to the BreadcrumbBar component that copies the full `_api/...` path to clipboard.

**Behavior (per user decisions):**
- Button appears **inline with the breadcrumb path text**, visible on hover over the breadcrumb bar area
- Copies the **full `_api/...` path** (REST endpoint path, not a URL with site prefix)
- Visual confirmation: **icon swap** — copy icon changes to checkmark for ~1.5s, then reverts
- Available on **any page that has a breadcrumb path** — the BreadcrumbBar always has at least `_api` segment

**Implementation:**

1. Add React imports: `useState, useCallback` from React.
2. Import icons from `lucide-react`: `Copy` and `Check`.
3. Compute the full API path from segments: join all segment labels with `/`. The first segment is always `_api`, so the result is `_api/web/lists/getbytitle(...)` etc.
4. Add state: `const [copied, setCopied] = useState(false)`.
5. Add copy handler:
   ```
   const handleCopy = useCallback(async () => {
     const apiPath = segments.map(s => s.label).join('/')
     await navigator.clipboard.writeText(apiPath)
     setCopied(true)
     setTimeout(() => setCopied(false), 1500)
   }, [segments])
   ```
6. Add the copy button **after** the last breadcrumb segment, inside the nav element. Position: `ml-2` margin from the last segment.
7. The button should be visible on hover over the breadcrumb bar area. Use a group hover pattern:
   - Add `group` class to the `<nav>` element
   - Button classes: `opacity-0 group-hover:opacity-100 transition-opacity` (plus standard button styling)
8. Button renders the `Copy` icon normally, swaps to `Check` icon with a green color when `copied` is true.
   - Normal: `<Copy className="h-3.5 w-3.5 text-muted-foreground" />`
   - Copied: `<Check className="h-3.5 w-3.5 text-green-500" />`
9. Add `aria-label="Copy API path"` and `title` showing what will be copied.

**Styling:** Small icon button, no border by default. `rounded p-1 hover:bg-accent transition-colors cursor-pointer`. Size should match the breadcrumb text height.
  </action>
  <verify>
Run `npx tsc -b --noEmit` — no errors. Navigate to any API endpoint (e.g., `/#/_api/web`). Hover over breadcrumb bar — copy button appears. Click copy button — icon changes to checkmark. Paste clipboard — contains `_api/web`.
  </verify>

<task type="auto">
  <name>Task 2: Create and wire up favicons</name>
  <files>app/public/favicon.svg, app/public/favicon.ico, app/index.html</files>
  <action>
Create proper favicons from the existing PNG assets and wire them into the HTML.

**Source images:** `app/src/assets/rest-explorer.png` (light mode, 192x192) and `app/src/assets/rest-explorer-dark.png` (dark mode, 192x192).

**Step 1: Create `app/public/favicon.svg`**

Create an SVG file that uses CSS `prefers-color-scheme` media query to switch between light and dark versions. Approach:

Since converting raster PNGs to vector SVG programmatically isn't practical, create a simple SVG favicon that represents the app brand. Use the app's color scheme — a simple design like the letters "SP" or a code bracket symbol `</>` in the app's primary colors.

Design a minimal SVG (32x32 viewBox):
- Light mode: dark icon on transparent background (matches light browser tabs)
- Dark mode (via `@media (prefers-color-scheme: dark)`): light icon on transparent background

Example approach — use a rounded rectangle with "SP" text or a bracket icon:
```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
  <style>
    .icon { fill: #1a1a2e; }
    @media (prefers-color-scheme: dark) {
      .icon { fill: #e2e8f0; }
    }
  </style>
  <rect width="32" height="32" rx="6" fill="#2563eb"/>
  <text x="16" y="22" text-anchor="middle" font-family="system-ui, sans-serif" font-size="14" font-weight="700" class="icon" fill="white">SP</text>
</svg>
```

Adjust the design to look good at 16x16 and 32x32 — keep it simple. Use the app's blue accent color (`#2563eb` or similar).

**Step 2: Create `app/public/favicon.ico`**

Use a Node.js script to generate a 32x32 ICO file from the light PNG:
```bash
node -e "
const sharp = require('sharp');
// ... resize to 32x32 and save as ICO
"
```

If `sharp` is not available, create a simple ICO-compatible PNG at 32x32 and rename it. Alternatively, use the SVG as the primary favicon and create a minimal ICO fallback.

Actually, the simplest approach: resize the light PNG to 32x32 as a PNG and use it as fallback. Modern browsers prefer SVG.

**Step 3: Update `app/index.html`**

Replace the current `<link rel="icon" type="image/svg+xml" href="/vite.svg" />` with:
```html
<link rel="icon" href="/favicon.svg" type="image/svg+xml" />
<link rel="icon" href="/favicon.ico" sizes="32x32" />
```

Remove the old `/vite.svg` reference. The `app/public/vite.svg` file can be deleted or left (it won't be referenced).
