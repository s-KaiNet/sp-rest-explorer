# Milestone v1.0: MVP

**Status:** SHIPPED 2026-02-12
**Phases:** 1-5
**Total Plans:** 11

## Overview

Rebuild the SharePoint REST API Metadata Explorer from Vue 2 + Webpack 3 to React 19 + Vite + Tailwind 4 + shadcn/ui. v1 scope is the Explore API view only — deep search, Explore Types, and API Changelog are deferred to v2. The 39 v1 requirements cluster into 5 natural delivery boundaries: scaffolding, data layer, navigation, explore views, and detail panels.

### Phase 2: Data Layer & UI Foundation

**Goal**: Metadata loads from Azure, is indexed for search, and the app displays loading/ready states with a consistent visual language.
**Depends on**: Phase 1 (build toolchain, routing shell)
**Plans**: 2 plans

Plans:
- [x] 02-01-PLAN.md — Data layer: TypeScript types, metadata singleton, Zustand store, lookup Maps, MiniSearch index, IndexedDB cache, boot orchestrator
- [x] 02-02-PLAN.md — UI foundation: color tokens, monospace font, CodeText component, skeleton screens, error state, CSS spinner, boot integration in App.tsx

**Requirements:** INFRA-03, INFRA-04, INFRA-05, INFRA-09, UIFN-01, UIFN-03, UIFN-04

**Details:**
- Frozen metadata singleton (4MB Object.freeze()) with useSyncExternalStore hook — outside Zustand to avoid deep-clone trap
- Zustand v5 status-only store (loading/error/ready) — zero metadata in reactive state
- Pre-computed lookup Maps: entityByFullName, functionById, entityChildren (sorted by kind)
- MiniSearch index with custom tokenizer (splits on dots/underscores for SP.Web-style names)
- IndexedDB cache via idb-keyval with cache-then-revalidate boot pattern
- OKLCH color tokens: blue (functions), green (entities), purple (nav properties) with dark mode variants
- CodeText component for inline monospace identifiers with variant coloring
- ContentSkeleton matching sidebar + content layout, ErrorState with retry, CSS spinner in index.html

## Milestone Summary

**Key Accomplishments:**
1. React 19 + Vite 7 + Tailwind CSS 4 + shadcn/ui project with GitHub Pages deployment pipeline
2. Frozen metadata singleton with O(1) lookup Maps, MiniSearch index, and IndexedDB cache-then-revalidate boot
3. Breadcrumb + resizable sidebar navigation with directional animations and composable function routing
4. Home screen with curated endpoint cards, browse-all with filtering, and recently visited tracking
5. Entity detail with collapsible Properties/NavProperties/Methods tables, TypeLink cross-references, and inline filters
6. Function detail with typed parameters, COMPOSABLE badges, and clickable entity type links

**Key Decisions:**
- MiniSearch over FlexSearch (native TS, returns full docs, faster init, simpler API)
- Sidebar as contextual nav, not tree (5-30 children per node, no virtualization needed)
- Metadata outside Zustand (frozen singleton + useSyncExternalStore prevents deep-clone perf trap)
- @tailwindcss/vite over PostCSS (Tailwind CSS 4 native plugin, better integration)
- OKLCH color space for type tokens (perceptually uniform, dark mode adjusts lightness only)
- Composable routing: composable functions resolve to entity children, non-composable are terminal
- TypeLink component for all entity references (consistent cross-linking everywhere)
- Contained scroll layout (h-screen + overflow-hidden, independent content/sidebar scroll)

**Issues Resolved:**
- shadcn CLI path alias resolution (root tsconfig.json needs paths for alias)
- Breadcrumb sticky inside overflow-hidden (changed to normal flex flow)
- TypeLink crash on undefined typeName from metadata gaps (added guard)
- Full-page scroll instead of contained (changed root to h-screen layout)
- Sidebar scroll leak to page level (absolute inset-0 containment)

**Issues Deferred:**
- NAV-03 (copy path button) — deferred, not implemented in v1
- Deep search (Cmd+K) — deferred to v2
- Explore Types full view — deferred to v2 (welcome screen only in v1)
- API Changelog view — deferred to v2
- GitHub Actions CI/CD — deferred to v2

**Technical Debt Incurred:**
- UsedByBar scans all entities on every render (no precomputed index) — acceptable for 2,449 entities, may need optimization at scale
- Recently visited kind mapping relies on depth heuristic (depth 2 = root) — fragile if tree structure changes
- Search placeholder UI shown but Cmd+K not functional — may confuse users expecting search
