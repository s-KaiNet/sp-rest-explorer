# Requirements Archive: SP REST API Explorer — New UI v1.3

**Archived:** 2026-02-17

## v1.3 Requirements — Improvements

### Search UX

- [x] **SRCH-05**: Typing a query containing dots or special characters (e.g., "SP.File") returns only results containing the literal substring — not split-token fuzzy matches — *validated v1.3, Phase 11 (11-01)*
- [x] **SRCH-06**: Clicking anywhere on a search result group header toggles collapse with a chevron indicator (▶/▼); collapsing/expanding causes zero layout shift — *validated v1.3, Phase 11 (11-01)*
- [x] **SRCH-07**: API Endpoints search results are sorted by path character length (shortest first) when a search query is active — *validated v1.3, Phase 11 (11-01)*
- [x] **SRCH-08**: Hovering over any search result item shows a visible background highlight, pointer cursor, and right-side arrow hint; works in both light and dark mode — *validated v1.3, Phase 11 (11-01, 11-02)*

### Detail Views

- [x] **DETL-01**: Properties with `nullable: false` in metadata show "no" in the Nullable column; all other properties (true, missing, undefined) show "yes" — *validated v1.3, Phase 12 (12-01)*

### Layout

- [x] **LAYT-01**: The Explore API breadcrumb renders inside the main content area (not spanning the sidebar), remains visible without sticky positioning (structurally above scroll container), and only appears on detail pages — *validated v1.3, Phase 12 (12-01, 12-02)*
