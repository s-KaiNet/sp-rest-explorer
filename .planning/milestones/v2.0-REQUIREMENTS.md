# Requirements Archive: SP REST API Explorer — v2.0 Backend Rework

**Archived:** 2026-02-24
**Status:** ✅ All 31 requirements complete

## v2.0 Requirements — All Complete

### Project Setup

- [x] **PROJ-01**: New `functions/` directory with Azure Functions v4 project structure *(Delivered as `backend/` directory)*
- [x] **PROJ-02**: TypeScript in strict mode with ESLint replacing deprecated TSLint
- [x] **PROJ-03**: All runtime dependencies installed and version-pinned

### Authentication

- [x] **AUTH-01**: Function authenticates to SharePoint using MSAL client credentials flow with certificate
- [x] **AUTH-02**: Authentication credentials configurable via environment variables

### Data Fetch

- [x] **FTCH-01**: Function fetches SharePoint `_api/$metadata` XML endpoint with Bearer token
- [x] **FTCH-02**: Fetch retries up to 3 times with exponential backoff on network failures and 5xx errors
- [x] **FTCH-03**: Fetch respects 429 Retry-After headers from SharePoint throttling
- [x] **FTCH-04**: Fetch has a 60-second timeout per attempt

### Data Processing

- [x] **PROC-01**: XML-to-JSON parsing produces identical output structure to legacy MetadataParser
- [x] **PROC-02**: TypeScript interfaces ported from legacy (all 7 types defined)
- [x] **PROC-03**: JSON output uses compact format (no indentation)
- [x] **PROC-04**: Parsed JSON is compressed via lz-string `compressToUTF16`

### Blob Storage

- [x] **BLOB-01**: Function uploads `metadata.latest.json` to `api-files` container
- [x] **BLOB-02**: Function uploads `metadata.latest.xml` to `api-files` container
- [x] **BLOB-03**: Function uploads `metadata.latest.zip.json` to `api-files` container
- [x] **BLOB-04**: Function uploads monthly snapshot `{year}y_m{month}_metadata.json` with 1-indexed months
- [x] **BLOB-05**: Function uploads monthly snapshot `{year}y_m{month}_metadata.xml` with 1-indexed months
- [x] **BLOB-06**: Function uploads monthly snapshot `{year}y_m{month}_metadata.zip.json` with 1-indexed months
- [x] **BLOB-07**: Container `api-files` is auto-created with public blob access level
- [x] **BLOB-08**: Blob uploads set correct Content-Type headers

### Operations

- [x] **OPS-01**: Function runs on a daily timer trigger (default 1 AM UTC)
- [x] **OPS-02**: Timer schedule is configurable via app setting
- [x] **OPS-03**: Date is computed fresh inside the function handler on every invocation
- [x] **OPS-04**: Function logs structured milestones with durations for each pipeline stage
- [x] **OPS-05**: HTTP trigger endpoint allows manual execution with function key authentication
- [x] **OPS-06**: Function-level retry policy configured for whole-function failures
- [x] **OPS-07**: If fetch fails after all retries, function exits without writing any blobs

### Deployment

- [x] **DEPL-01**: `npm run build` compiles TypeScript to JavaScript
- [x] **DEPL-02**: `npm run deploy` builds and publishes to Azure
- [x] **DEPL-03**: `.funcignore` excludes local.settings.json, source maps, tests, and source TypeScript

## Out of Scope (Documented)

| Feature | Reason |
|---------|--------|
| Diff generation (GenerateDiff function) | Diffs computed client-side with jsondiffpatch |
| `diff-files` blob container | No server-side diffs |
| Weekly snapshots | Never consumed by frontend |
| Client secret auth | SharePoint Online blocks client secrets for app-only access |
| Durable Functions orchestration | Pipeline takes ~2.5s — overkill |
| Multi-tenant support | Single SharePoint tenant |
| Azure Key Vault for certificates | *(Ended up using Key Vault anyway — app settings mangle PEM)* |
| CI/CD in v2.0 | Premature — deferred to ADDL-02 |
| Frontend changes | Separate future milestone |
| Modifying legacy `az-funcs/` | Clean-room rewrite in `backend/` |
