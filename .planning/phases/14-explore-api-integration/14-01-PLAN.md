  <files>app/src/components/navigation/SidebarItem.tsx</files>
  <action>
Rewrite SidebarItem to use TypeIcon instead of text badges. Per user decisions:

1. **Remove `showTypeTags` prop entirely** ‚Äî icons are always visible, not optional.
2. **Remove the `variant` prop** ‚Äî root and child entries share the same structural layout; only the icon type differs.
3. **Add a new `apiType` prop** of type `ApiType` (from `@/lib/api-types`) ‚Äî the caller determines the type (since `ChildEntry.kind` doesn't include 'root', the caller passes 'root' | 'function' | 'nav').
4. **Icon-first layout:** Render `<TypeIcon type={apiType} size="sm" />` as the FIRST child in the button flex row, BEFORE the label text span. The TypeIcon carries the type color; the label stays default text color.
5. **Remove ALL badge spans** ‚Äî the root `<>` badge, `FN` badge, and `NAV` badge. Replace them ALL with the single TypeIcon at the start.
6. **Keep existing button classes** for hover/sizing (`flex w-full cursor-pointer items-center gap-2 rounded-md px-2 py-1.5 text-left text-sm transition-colors hover:bg-accent`).
7. **Keep `displayName` prop** and the `title` attribute on the button.
8. **Icon color does NOT change on hover/selection** ‚Äî no special hover classes on TypeIcon.
9. **Bare icon, no container** ‚Äî no background pill, circle, or square around the icon.

The mapping from the caller side will be:
- Root variant entries ‚Üí `apiType="root"` (green Box)
- `entry.kind === 'function'` ‚Üí `apiType="function"` (blue Zap)
- `entry.kind === 'navProperty'` ‚Üí `apiType="nav"` (purple Compass)

Final props interface should be: `{ entry: ChildEntry; onClick: () => void; apiType: ApiType; displayName?: string }`.

Import `TypeIcon` from `@/components/ui/type-icon` and `ApiType` from `@/lib/api-types`.
