  <action>
    Modify `App.tsx` to:

    1. **Import CommandPalette** from `@/components/search`
    2. **Add palette open state:** `const [paletteOpen, setPaletteOpen] = useState(false)`
    3. **Add global Cmd+K / Ctrl+K keyboard shortcut:**
       ```tsx
       useEffect(() => {
         const handleKeyDown = (e: KeyboardEvent) => {
           if ((e.metaKey || e.ctrlKey) && e.key === 'k') {
             e.preventDefault()
             // Only open if metadata is ready (per decision: "Palette disabled until metadata is loaded")
             const status = useAppStore.getState().status
             if (status !== 'ready') return
             setPaletteOpen((prev) => !prev) // Toggle behavior per decision
           }
         }
         document.addEventListener('keydown', handleKeyDown)
         return () => document.removeEventListener('keydown', handleKeyDown)
       }, [])
       ```

    4. **Handle result selection:**
       - Import `useNavigate` from `react-router` and `useRecentlyVisited` from `@/hooks`
       - On select, receive the path string from CommandPalette
       - Navigate to the selected path: `navigate(path)`
       - Add to recently visited: determine `name` (last path segment), `kind` (parse from path or pass from CommandPalette), and call `addVisit({ name, path, kind })`
       - Close the palette: `setPaletteOpen(false)`
       - **Refinement:** Rather than parsing kind from path, modify the `onSelect` callback to receive `{ path: string, name: string, kind: 'entity' | 'function' | 'navProperty' }` so CommandPalette passes all needed info.

    5. **Pass palette trigger to Header and HomePage:**
       Since App.tsx renders `<Header />` directly and pages render via `<Outlet />`, we need a way for Header and HomePage to open the palette.
       **Approach — use a shared callback via React context or prop drilling through Outlet context:**
       - Use `useOutletContext` from react-router to pass `onSearchClick` to child routes
       - Pass `onSearchClick` directly to Header as a prop (Header is rendered directly in App, not via Outlet)
       - Define: `const onSearchClick = () => { if (useAppStore.getState().status === 'ready') setPaletteOpen(true) }`
       - Pass to Header: `<Header onSearchClick={onSearchClick} />`
       - Pass to Outlet: `<Outlet context={{ onSearchClick }} />`

    6. **Render CommandPalette** alongside the rest of the UI (inside the main div but outside the conditional rendering):
       ```tsx
       <CommandPalette
         open={paletteOpen}
         onOpenChange={setPaletteOpen}
         onSelect={handleSelect}
       />
       ```
       Place it after `</main>` but inside the outer div, so it renders regardless of app status.

    7. **Clear query on open:** The CommandPalette should reset its query when `open` changes to `true` (handle this inside CommandPalette itself via a `useEffect` on `open` prop). Note this for Plan 01 executor — if not already handled, add it here as a prop or document the expectation.
