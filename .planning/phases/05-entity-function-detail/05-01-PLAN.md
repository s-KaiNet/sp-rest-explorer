  <action>
    Create the shared building-block components for entity detail:

    **TypeLink.tsx** — Clickable entity type link component. Per user decision:
    - ALL entity type links navigate to `/#/entity/{typeName}` using React Router `Link`.
    - `Collection(SP.ListItem)` renders as split link: "Collection" in muted/secondary color linking to `/#/entity/Collection(SP.ListItem)`, parentheses as plain text, "SP.ListItem" in entity-green linking to `/#/entity/SP.ListItem`. Both get underline-on-hover.
    - `Collection(Edm.*)` types: NOT linkable — render as plain gray/muted monospace text.
    - Primitive types (`Edm.String`, `Edm.Int32`, `Edm.Boolean`, `Edm.DateTime`, `Edm.Guid`, `Edm.Binary`, `Edm.Byte`, `Edm.Double`, `Edm.Single`, `Edm.Int16`, `Edm.Int64`, `Edm.Stream`, `Edm.Time`): gray/muted monospace text, NOT clickable.
    - All clickable entity type links get hover tooltip: "View {typeName} entity". For collection split links: Collection tooltip = "View Collection(SP.ListItem) entity", inner type tooltip = "View SP.ListItem entity".
    - Props: `typeName: string` (the full type string like "SP.List" or "Collection(SP.ListItem)" or "Edm.String").
    - Parse Collection pattern: regex `/^Collection\((.+)\)$/` to detect and extract inner type.
    - Detect primitive: starts with "Edm." (after extraction from Collection).

    **CollapsibleSection.tsx** — Collapsible section wrapper. Per user decision:
    - Sections with 0 items: collapsed by default, (0) count badge. When expanded: show italic message (passed as `emptyMessage` prop).
    - Sections with items: expanded by default.
    - Props: `title: string`, `count: number`, `emptyMessage?: string`, `filterSlot?: ReactNode` (for inline filter), `children: ReactNode`.
    - Renders section header with h3 title, count badge (gray bg rounded pill per mockup `.detail-section-count`), optional filter slot, and chevron (▲ when expanded, ▼ when collapsed).
    - Header is clickable to toggle. Use `cursor-pointer select-none` on header.
    - Section header has `border-b-2 border-border` bottom border per mockup.
    - ID prop for scroll targeting by SectionJumpLinks.

    **SectionFilter.tsx** — Inline filter input for Properties and Methods sections. Per mockup `.section-filter`:
    - Small search input (160px, expands to 200px on focus) with search icon.
    - Props: `value: string`, `onChange: (value: string) => void`, `placeholder: string`.
    - Positioned in section header via flexbox (margin-left: auto).

    **SectionJumpLinks.tsx** — Section jump pill links. Per mockup `.section-jump`:
    - Three pills: "Properties (N)", "Navigation Properties (N)", "Methods (N)".
    - Each has a count badge in gray bg. Per user decision: show all three always including (0) — dimmed/muted styling for empty sections.
    - Click scrolls to section ID with smooth scroll.
    - Props: `propertiesCount: number`, `navPropertiesCount: number`, `methodsCount: number`.

    **BaseTypeChain.tsx** — Base type inheritance chain. Per mockup `.base-type-chain`:
    - Gray background rounded box with "Base type:" label.
    - Chain of entity type links separated by → arrows.
    - Walk the baseTypeName chain: given an entity, look up its baseTypeName, then that entity's baseTypeName, etc.
    - Uses TypeLink for each base type (so they're clickable green links navigating to `/#/entity/{baseTypeName}`).
    - If entity has no baseTypeName, don't render this component.
    - Props: `entity: EntityType`. Needs access to `useLookupMaps()` to resolve baseTypeName chain.

    **UsedByBar.tsx** — "Used by" cross-reference bar. Per user decision:
    - Purple background bar (matching mockup `.used-by` with `bg-type-nav/10` or similar purple-light).
    - Label: "Referenced as navigation property in:".
    - Shows chips like "SP.Web .Lists" — entity name + dot + property name.
    - Per user decision: scope = navigation properties only (NOT regular properties).
    - Computation: on-demand per render. Scan all entities' nav properties to find which reference this entity's fullName. Use `useMetadataSnapshot()` to access all entities.
    - Check both direct reference (`typeName === entity.fullName`) and Collection reference (`typeName === "Collection(entity.fullName)"`).
    - Chips are clickable links: navigate to `/#/entity/{referencingEntityFullName}`.
    - Per user decision: show first 10 chips, then "+N more" expandable pill. Hide entirely when 0 references.
    - Chip styling per mockup: `.used-by-chip` — mono font, purple bg with border on hover.
    - Props: `entityFullName: string`.

    **index.ts** — Barrel export: `TypeLink`, `EntityDetail`, `CollapsibleSection`, `SectionJumpLinks`, `BaseTypeChain`, `UsedByBar` (and all table components from Task 2).

    - Sorted alphabetically by property name (always, per user decision — no interactive sort toggles).
    - No pagination, no virtualization — render all rows.
    - Props: `properties: Property[]` (already sorted or sort internally).

    **NavPropertiesTable.tsx** — Navigation properties table. Per mockup:
    - Columns: Name (40%) | Target Type (60%).
    - Same header styling as PropertiesTable.
    - Name column: monospace font-medium in purple (`text-type-nav`) per ENTD-10 and mockup `.nav-prop-name`.
    - Target Type: use `TypeLink` component (handles Collection split links).
    - Sorted alphabetically by name.
    - Props: `navigationProperties: NavigationProperty[]`.

    **MethodsTable.tsx** — Methods table. Per mockup `.methods-table`:
    - Columns: Method (25%) | Parameters (48%) | Returns (27%).
    - Method name: blue monospace font-semibold (`text-type-fn font-mono font-semibold`) per ENTD-11 and mockup `.method-name-cell`.
    - Parameters column: one-per-line as `paramName: ParamType` per ENTD-06. Use `<ul>` with no list style. Each param: `.p-name` (foreground, font-medium) + ": " + `.p-type` (use TypeLink for entity types, muted text for Edm.*). Filter out `this` parameter (per user decision, already in current code). When no params (or only `this`): show italic muted "none" text per mockup `.no-params`.
    - Returns column: use TypeLink for entity types. `void` return type: italic muted text per mockup. If `isComposable`: show `COMPOSABLE` badge next to return type (inline-block, small blue bg pill, per mockup `.badge-composable`).
    - Sorted alphabetically by method name.
    - Per user decision: `this` parameter filtered from display.
    - vertical-align: top on all `<td>` per mockup (params can be multi-line).
    - Props: `functions: FunctionImport[]` (the entity's bound functions, resolved from `functionIds`).

    **EntityDetail.tsx** — Main orchestrator component:
    - Props: `entity: EntityType` (the entity to display).
    - Layout (top to bottom, per mockup `.detail-content` with `p-6`):
      1. **Header**: Entity name (22px bold) + "Entity Type" badge (green-light bg, entity-green text, rounded).
      2. **Full name**: `entity.fullName` in secondary mono text with gray code bg.
      3. **BaseTypeChain**: if entity has baseTypeName.
      4. **UsedByBar**: if entity has cross-references (component handles hiding if 0).

      8. **Methods section**: CollapsibleSection wrapping MethodsTable, with SectionFilter in filterSlot. Filter state managed here. Resolve `entity.functionIds` to `FunctionImport[]` via `useLookupMaps()`.

    - Empty section messages per user decision: "No properties" / "No navigation properties" / "No methods".
    - All sections use IDs matching SectionJumpLinks targets: `props-section`, `navprops-section`, `methods-section`.
