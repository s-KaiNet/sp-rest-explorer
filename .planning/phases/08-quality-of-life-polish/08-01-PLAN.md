  <files>app/src/lib/github.ts, app/src/components/layout/Header.tsx</files>
  <action>
**Create `app/src/lib/github.ts`:**

Export an async function `fetchStarCount(repo: string): Promise<number | null>` that:
1. Checks localStorage for cached value. Cache key: `sp-explorer-gh-stars`. Stored as JSON: `{ count: number, fetchedAt: number }`.
2. If cached AND `Date.now() - fetchedAt < 30 * 24 * 60 * 60 * 1000` (30 days), return cached count immediately.
3. Otherwise, fetch from `https://api.github.com/repos/${repo}` (no auth needed for public repos).
4. Extract `stargazers_count` from response JSON.
5. Store in localStorage with current timestamp.
6. Return the count. On any error (network, rate limit), return cached value if available, else null.

**Update `app/src/components/layout/Header.tsx`:**

Add star count display near the GitHub link in the header's right section.

1. Import `fetchStarCount` from `@/lib/github`.
2. Add state: `const [starCount, setStarCount] = useState<number | null>(null)`.
3. Add useEffect that calls `fetchStarCount('s-KaiNet/sp-rest-explorer')` on mount and sets state.
4. Display the star count next to the GitHub icon. Integration approach: modify the GitHub `<a>` to include a small star count badge when `starCount !== null`.
   - Show as: `★ {formatted_count}` (e.g., "★ 234") to the left of or integrated with the GitHub icon
   - Format: if count >= 1000, show as "1.2k" etc.
   - Style: `text-xs text-muted-foreground` — subtle, not a separate badge. The star count should feel "nicely integrated" with the GitHub link (per user decision).
   - Layout: the `<a>` element wraps both the star text and the GitHub icon together so they act as one clickable unit.
