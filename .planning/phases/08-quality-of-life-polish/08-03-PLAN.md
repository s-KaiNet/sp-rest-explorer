<task type="auto">
  <name>Task 1: Rethink dark mode colors (GitHub Dark inspired)</name>
  <files>app/src/index.css</files>
  <action>
Update the `.dark { }` block in `app/src/index.css` to use colors inspired by the VS Code GitHub Dark theme. The goal is **milder dark mode** — less harsh contrast, more muted tones.

**Reference:** VS Code GitHub Dark theme (visible at https://github.dev). Key characteristics:
- Background: NOT pure black — a dark gray with slight blue undertone (~#0d1117 or ~#24292e)
- Foreground: NOT pure white — a soft off-white (~#c9d1d9 or ~#e6edf3)
- Borders: subtle, low-contrast (~#30363d)
- Muted text: medium gray (~#8b949e)
- Sidebar: slightly different shade from main content
- Accent/selection: subtle blue tint

**Convert to oklch (the format already used in index.css).** Use approximate conversions:

Replace ALL dark mode variables. Suggested mapping (adjust as needed for visual quality):

```css
.dark {
  /* GitHub Dark: #0d1117 (main bg) */
  --background: oklch(0.14 0.005 260);
  /* GitHub Dark: #c9d1d9 (primary text) */
  --foreground: oklch(0.85 0.01 250);
  /* Card same as background or very slight lift */
  --card: oklch(0.14 0.005 260);
  --card-foreground: oklch(0.85 0.01 250);
  /* Popover slightly lifted */
  --popover: oklch(0.18 0.005 260);
  --popover-foreground: oklch(0.85 0.01 250);
  /* Primary: soft white */
  --primary: oklch(0.85 0.01 250);
  --primary-foreground: oklch(0.14 0.005 260);
  /* Secondary: #161b22 */
  --secondary: oklch(0.20 0.005 260);
  --secondary-foreground: oklch(0.85 0.01 250);
  /* Muted: #161b22 bg, #8b949e text */
  --muted: oklch(0.20 0.005 260);
  --muted-foreground: oklch(0.62 0.01 250);
  /* Accent: #161b22 */
  --accent: oklch(0.20 0.005 260);
  --accent-foreground: oklch(0.85 0.01 250);
  /* Destructive unchanged */
  --destructive: oklch(0.577 0.245 27.325);
  --destructive-foreground: oklch(0.577 0.245 27.325);
  /* Border: #30363d */
  --border: oklch(0.30 0.005 260);
  --input: oklch(0.30 0.005 260);
  /* Ring: slightly brighter border */
  --ring: oklch(0.45 0.01 250);
  /* Sidebar: #010409 or #0d1117 slightly darker */
  --sidebar-background: oklch(0.12 0.005 260);
  --sidebar-foreground: oklch(0.85 0.01 250);
  --sidebar-primary: oklch(0.85 0.01 250);
  --sidebar-primary-foreground: oklch(0.12 0.005 260);
  --sidebar-accent: oklch(0.20 0.005 260);
  --sidebar-accent-foreground: oklch(0.85 0.01 250);
  --sidebar-border: oklch(0.30 0.005 260);
  --sidebar-ring: oklch(0.45 0.01 250);
  /* Type colors — keep vibrant but adjust for new backgrounds */
  --type-fn: oklch(0.72 0.16 255);
  --type-entity: oklch(0.68 0.14 155);
  --type-nav: oklch(0.72 0.16 300);
  /* Code blocks */
  --code-bg: oklch(0.18 0.005 260);
}
```

**Key principles:**
- Slight blue undertone (hue ~260) in grays — this is the GitHub Dark signature
- Background ~0.14 lightness (not 0.145 pure neutral — add the blue tint)
- Foreground ~0.85 (not 0.985 pure white — softer)
- Muted foreground ~0.62 (readable but clearly secondary)
- Borders at ~0.30 lightness (subtle but visible)
- Sidebar slightly darker than main content
- Type colors adjusted to be readable on the new background

**IMPORTANT:** Only modify the `.dark { }` block. Do NOT change `:root { }` (light mode) or `@theme inline { }` or any other CSS.

<task type="auto">
  <name>Task 2: Add dark mode scrollbar styling</name>
  <files>app/src/index.css</files>
  <action>
Add global dark scrollbar styling to `app/src/index.css` that applies to **all scrollable areas** in dark mode (sidebar, content panels, modals, etc.).

**Approach:** Use both `scrollbar-color` (standards-based, Firefox) and `::-webkit-scrollbar` (Chrome/Edge/Safari) for cross-browser coverage.

Add the following CSS at the end of the file (after the existing `body, header, main { transition: ... }` block):

```css
/* Dark mode scrollbars — global, applies to all scrollable areas */
.dark {
  scrollbar-color: oklch(0.35 0.005 260) transparent;
  scrollbar-width: thin;
}

.dark ::-webkit-scrollbar {
  width: 8px;
  height: 8px;
}

.dark ::-webkit-scrollbar-track {
  background: transparent;
}

.dark ::-webkit-scrollbar-thumb {
  background: oklch(0.35 0.005 260);
  border-radius: 4px;
}

.dark ::-webkit-scrollbar-thumb:hover {
  background: oklch(0.42 0.005 260);
}

.dark ::-webkit-scrollbar-corner {
  background: transparent;
}
```

**Color choices:**
- Thumb color: slightly lighter than border (~0.35 lightness with blue tint), matching the GitHub Dark theme variables from Task 1
- Track: transparent (the background shows through)
- Hover: slightly brighter (~0.42)
- Width: `thin` for Firefox, `8px` for WebKit (slim but grabable)

**Per user decision:** Apply globally to ALL scrollable areas — the `.dark` selector on `<html>` makes descendants inherit. No need to target specific components.

Do NOT add any light mode scrollbar overrides — keep the system default for light mode.
  </action>

  </verify>
