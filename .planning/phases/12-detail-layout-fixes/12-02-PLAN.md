**ExplorePage.tsx — restructure the right content panel (lines 126-230):**

The current structure is:
```
<div className="flex-1 overflow-y-auto bg-muted/30">   ← single scroll container
  <BreadcrumbBar ... />                                  ← scrolls with content (sticky hack)
  <ContentTransition ...>                                ← content
```

Change to a flex column layout where breadcrumb is OUTSIDE the scroll container:
```
<div className="flex flex-1 flex-col overflow-hidden bg-muted/30">   ← flex column, no scroll here
  {!isRoot && segments.length > 0 && (
    <BreadcrumbBar segments={segments} onNavigate={handleBreadcrumbNavigate} />
  )}                                                                  ← breadcrumb: non-scrolling, shrink-0
  <div className="flex-1 overflow-y-auto">                            ← NEW inner scroll wrapper
    <ContentTransition pathKey={pathKey}>
      ...existing content...
    </ContentTransition>
  </div>
</div>
```

Specifically:
1. Change line 127 outer div from `"flex-1 overflow-y-auto bg-muted/30"` to `"flex flex-1 flex-col overflow-hidden bg-muted/30"` — makes it a flex column that does NOT scroll itself.
2. Keep the BreadcrumbBar conditional render (lines 128-131) exactly where it is — it becomes a natural non-scrolling flex child.
3. Wrap the `<ContentTransition>` (and its closing tag) in a new `<div className="flex-1 overflow-y-auto">` — this div becomes the ONLY scroll container.

**BreadcrumbBar.tsx — remove sticky, add shrink-0:**

On line 25, the nav element has class `"group sticky top-0 z-10 flex min-h-[40px] flex-wrap items-center border-b border-border bg-sidebar px-4 py-2"`.

Remove `sticky top-0 z-10` from the className — no longer needed since the breadcrumb is outside the scroll container.
Add `shrink-0` to prevent the breadcrumb from shrinking when flex column gets tight.

The updated className should be:
`"group flex shrink-0 min-h-[40px] flex-wrap items-center border-b border-border bg-sidebar px-4 py-2"`

These two changes together ensure:
- Breadcrumb stays fixed at top of content area (it's a flex child above the scroll container)
- Scrollbar only appears for the content below the breadcrumb
- No transparency artifacts because content never scrolls behind the breadcrumb
