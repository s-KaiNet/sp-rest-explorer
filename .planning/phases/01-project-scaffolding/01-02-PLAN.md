  <action>
    **Theme System:**

    Create `app/src/components/theme/ThemeProvider.tsx` implementing a React context-based theme provider:
    - On mount, check localStorage for `"sp-explorer-theme"` key
    - If no stored value exists, detect system preference via `window.matchMedia('(prefers-color-scheme: dark)')` and use it as initial theme
    - If a stored value exists, use it (ignoring system preference per user decision)
    - Toggle function switches between "light" and "dark", saves to localStorage, and adds/removes the `dark` class on `document.documentElement`
    - Export `ThemeProvider` component and `useTheme` hook (returns `{ theme, toggleTheme }`)
    - The provider must set the initial class on `<html>` synchronously to avoid flash of wrong theme (use a script block in index.html or useLayoutEffect)

    Create `app/src/components/theme/DarkModeToggle.tsx`:
    - Import `Sun` and `Moon` icons from `lucide-react`
    - Use `useTheme()` hook
    - Render a button: shows Sun icon in dark mode (clicking switches to light), Moon icon in light mode (clicking switches to dark)
    - Style as a ghost-style icon button (transparent bg, subtle hover state) — use Tailwind classes directly (no shadcn Button dependency yet)
    - Two states only: Light ↔ Dark, no "system" option in the toggle (per user decision)

    Create barrel export `app/src/components/theme/index.ts` re-exporting `ThemeProvider`, `useTheme`, and `DarkModeToggle`.

    **Route Configuration:**

    Create `app/src/routes.tsx` with `createHashRouter` from `react-router`:
    ```typescript
    import { createHashRouter } from 'react-router'
    ```

    Define routes matching the current URL patterns from the old Vue router:
    - `/` — ExplorePage (home/Explore API — index route)
    - `/_api/*` — ExplorePage (API browse with catch-all path for deep linking)
    - `/entity` — TypesPage (entity list)
    - `/entity/:typeName` — TypesPage (entity detail)
    - `/entity/:typeName/func/:funcName` — TypesPage (function on entity)
    - `/api-diff` — ChangelogPage
    - `/api-diff/:monthKey` — ChangelogPage (monthly changelog)
    - `/how-it-works` — HowItWorksPage
    - `*` — NotFoundPage (catch-all 404 for unknown routes)

    Use `App` as the root layout element wrapping an `<Outlet />`. All routes are children of the root layout route.

    Export the router instance as `router`.

    **Page Components:**

    Create placeholder page components in `app/src/pages/`:

    `ExplorePage.tsx` — For now, render a simple centered message: "SP REST API Explorer" with a brief subtitle. This is the home page and will be replaced with real content in Phase 4.

    `TypesPage.tsx`, `ChangelogPage.tsx`, `HowItWorksPage.tsx` — Each renders a centered "Coming soon" message. Same generic design for all three (per user decision). Use the page/route name as a small heading above the message so users know which page they're on.

    `NotFoundPage.tsx` — Renders a centered "Page not found" message (per user decision). Include a link back to home (`/`).

    All placeholder pages: vertically and horizontally centered content using Tailwind flex utilities. Use `text-muted-foreground` for the message text so it's subtle.

    Create barrel export `app/src/pages/index.ts`.

    **Wire it all together in main.tsx:**

    Update `app/src/main.tsx`:
    ```tsx
    import { RouterProvider } from 'react-router'
    import { ThemeProvider } from '@/components/theme'
    import { router } from '@/routes'

    // Render:
    <ThemeProvider>
      <RouterProvider router={router} />
    </ThemeProvider>
    ```

    Update `app/src/App.tsx` to be the root layout:
    - Import `Outlet` from `react-router`
    - Render `<Header />` at the top and `<Outlet />` below for route content
    - The main content area should have `pt-14` (or similar) to account for the fixed header
    - Wrap in `min-h-screen bg-background text-foreground`

    **Anti-flash theme script:**
    Add an inline script to `app/index.html` in the `<head>` that reads localStorage and applies the `dark` class before React hydrates:
    ```html
    <script>
      (function() {
        var stored = localStorage.getItem('sp-explorer-theme');
        if (stored === 'dark' || (!stored && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
          document.documentElement.classList.add('dark');
        }
      })();
    </script>
    ```
    This prevents the flash of light theme on dark-mode users.
  </action>
