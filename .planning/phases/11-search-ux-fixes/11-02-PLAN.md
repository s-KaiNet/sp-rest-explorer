  <action>
Two files need changes:

**1. `app/src/components/ui/command.tsx` — CommandItem base class (line 158)**

In the CommandItem component's className string, replace:
- `data-[selected=true]:bg-accent` → `data-[selected=true]:bg-foreground/8`

This changes the keyboard-selection highlight from the invisible `--accent` color to an 8% foreground overlay, which is always visible regardless of `--popover` background.

**2. `app/src/components/search/CommandPalette.tsx` — Two CommandItem usages**

In `renderEntityItem` (line 347) and `renderEndpointItem` (line 375), replace:
- `hover:bg-accent` → `hover:bg-foreground/5`

This changes the mouse-hover highlight from the invisible `--accent` to a 5% foreground overlay. The hover is intentionally subtler (5%) than keyboard selection (8%) to differentiate the two states.

**Why foreground/N instead of fixing --accent in CSS:**
Changing `--accent` globally would affect every component using that token (buttons, menus, etc.). A targeted fix on the two affected components is safer and more intentional. Semi-transparent foreground overlays are theme-resilient by definition — they always contrast with whatever background they sit on.
