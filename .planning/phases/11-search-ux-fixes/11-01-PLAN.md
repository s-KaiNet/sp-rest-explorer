**Fix SearchGroup collapse UX (SRCH-07):**

The current `SearchGroup` component (lines 145-210) has two problems:
1. When collapsed, it renders a completely different DOM structure (a plain div) than when expanded (a CommandGroup with heading). This causes layout shift.
2. The collapse toggle is a small button inside the heading — not the full header row.

Redesign the `SearchGroup` component. The approach: render our own header div above a `<CommandGroup>` that has NO heading prop. This way cmdk keyboard navigation still works for items, while the header is fully controlled and consistent.

1. **Render a standalone header div** above the CommandGroup:
   - A chevron indicator: `▶` (Unicode 25B6) when collapsed, `▼` (Unicode 25BC) when expanded
   - The heading text (e.g., "Entities")
   - The count badge (e.g., "(42)")
   - The entire row is the click target with `cursor-pointer`
   - Use `px-2 py-1.5 text-xs font-medium text-muted-foreground` matching existing cmdk group-heading styles
   - `onClick={() => setCollapsed(!collapsed)}` on the header div — no separate small button

2. **When collapsed:** Render only the header div. Do NOT render the CommandGroup or items at all.
   **When expanded:** Render the header div + a `<CommandGroup>` (without heading prop) containing the visible `<CommandItem>` children.

3. **"Show more" button:** Render as a plain `div` (not CommandItem) to avoid cmdk interference.

4. Use consistent padding/height for the header in BOTH states so text never shifts position.

Implementation structure:
```tsx
function SearchGroup({ heading, results, expanded, setExpanded, collapsed, setCollapsed, renderItem }) {
  if (results.length === 0) return null
  const visible = (!collapsed && expanded) ? results : (!collapsed ? results.slice(0, INITIAL_SHOW) : [])
  const remaining = results.length - INITIAL_SHOW

  return (
    <div className="p-1">
      {/* Header row — always rendered identically */}
      <div
        className="flex items-center gap-1.5 px-2 py-1.5 text-xs font-medium text-muted-foreground cursor-pointer select-none hover:text-foreground"
        onClick={() => setCollapsed(!collapsed)}
      >
        <span className="w-3 text-center">{collapsed ? '\u25B6' : '\u25BC'}</span>
        <span>{heading}</span>
        <span>({results.length})</span>
      </div>
      {/* Items in CommandGroup (no heading) — hidden when collapsed */}
      {!collapsed && (
        <CommandGroup>
          {visible.map(renderItem)}
          {remaining > 0 && !expanded && (
            <div
              className="flex justify-center px-2 py-1.5 text-xs text-muted-foreground cursor-pointer hover:text-foreground"
              onClick={() => setExpanded(true)}
            >
              Show {remaining} more…
            </div>
          )}
        </CommandGroup>
      )}
    </div>
  )
}
```

Items remain `<CommandItem>` inside `<CommandGroup>` so arrow-key navigation works. Only the header is a plain div.

Per user decision: API Endpoints results sorted by path character length, shortest first. Only applies when there's an active search query. Ties preserve original order (stable sort).

In `CommandPalette.tsx`, find the `endpoints` useMemo (around lines 299-305):

```tsx
const endpoints = useMemo(
  () =>
    searchMode === 'path'
      ? (searchResults ?? [])
      : (searchResults ?? []).filter((r) => r.kind === 'endpoint'),
  [searchResults, searchMode],
)
```

Add path-length sorting AFTER filtering. Only sort when `debouncedQuery` is active (length >= 3):

```tsx
const endpoints = useMemo(() => {
  const filtered = searchMode === 'path'
    ? (searchResults ?? [])
    : (searchResults ?? []).filter((r) => r.kind === 'endpoint')

  // SRCH-08: Sort by path length (shortest first) when searching
  if (debouncedQuery.length >= 3 && filtered.length > 0) {
    return [...filtered].sort((a, b) => {
      const pathA = (a.path as string) ?? ''
      const pathB = (b.path as string) ?? ''
      return pathA.length - pathB.length
    })
  }

  return filtered
}, [searchResults, searchMode, debouncedQuery])
```

**Key points per user decision:**
- Sort metric is character count of the path STRING (not segment count)
- Use `[...filtered].sort()` to avoid mutating the original array. `Array.sort` is stable in all modern browsers (ES2019+), so ties preserve original order.
- Only sort when `debouncedQuery.length >= 3` — no query means no path-length sort
- This applies to BOTH name-mode and path-mode endpoint results
