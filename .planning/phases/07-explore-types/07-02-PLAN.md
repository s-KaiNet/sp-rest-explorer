**TypeLink.tsx update — complex type awareness:**

Per CONTEXT.md decisions:
- "Clicking a TypeLink in Explore API navigates to Explore Types AND syncs the sidebar to highlight/scroll to that type"
- "TypeLinks within Explore Types (on property types): complex types navigate within Explore Types; entity type references navigate to Explore API"

The current TypeLink navigates ALL non-Edm types to `/entity/{typeName}`. Since both complex types and entity types already live under the `/entity/` route, and TypesPage (from Task 2) already handles rendering the right detail view based on whether the type is complex or entity, **TypeLink navigation doesn't need to change the URL pattern** — it already works correctly!

The sidebar sync (scroll-to-active on TypesPage) is already handled by Task 2's scroll-into-view effect. When a TypeLink is clicked within Explore Types, it navigates to `/entity/{typeName}`, the TypesPage re-renders with the new param, and the sidebar scrolls to show the selected type.

However, there IS one subtle change needed: TypeLinks for entity types clicked WITHIN Explore Types should ideally navigate to Explore API (the `/_api/` route tree) rather than staying on `/entity/`. But per the current route structure, entity types on `/entity/:typeName` already render with `EntityDetail` and work correctly. The CONTEXT.md says entity type references should "navigate to Explore API" — but there's no clean mapping from an entity fullName to an `_api/` path (entities exist at various depths in the tree).

**Decision:** Keep all TypeLinks navigating to `/entity/{typeName}`. TypesPage already handles both complex and entity types at this URL. The key UX wins (sidebar sync, complex type detail) are already achieved. Navigating entity types to Explore API would require a complex fullName→path resolver that doesn't exist and isn't worth building for this phase.

**What to actually change in TypeLink.tsx:**
- Import `getTypeIndexes` from `@/lib/metadata` (non-React — TypeLink is called frequently, avoid hook overhead)
- For non-Edm, non-Collection types: check `getTypeIndexes()?.complexTypeNames.has(typeName)`
  - If complex type: keep `title` as `"View ${typeName} type"` (not "entity")
  - If entity type: keep existing `title` as `"View ${typeName} entity"`
- This is a title-only change. Navigation URL stays the same (`/entity/${typeName}`).
