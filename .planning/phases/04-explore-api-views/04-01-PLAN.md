  <files>app/src/hooks/use-recently-visited.ts, app/src/hooks/index.ts, app/src/pages/ExplorePage.tsx</files>
  <action>
Create `app/src/hooks/use-recently-visited.ts` with:

**Types:**
```ts
interface RecentlyVisitedItem {
  name: string        // Display name (e.g. "GetByTitle")
  path: string        // Full hash path (e.g. "/_api/web/Lists/GetByTitle")
  kind: 'function' | 'navProperty' | 'root'  // For icon coloring
  timestamp: number   // Date.now() when visited
}
```

**Hook `useRecentlyVisited()`** returns:
- `items: RecentlyVisitedItem[]` — sorted by most recent first, max 12 items
- `addVisit(item: Omit<RecentlyVisitedItem, 'timestamp'>): void` — adds/bumps item to top
- `clearAll(): void` — clears all history

**Implementation details:**
- localStorage key: `'recently-visited'`
- On `addVisit`: if item with same `path` exists, update timestamp; otherwise prepend. Trim to 12 max. Write to localStorage.
- Initialize from localStorage on mount (parse with try/catch, default to empty array on error)
- Use `useState` for reactive updates within the session
- Export the `RecentlyVisitedItem` type and `useRecentlyVisited` hook

**Wire into ExplorePage.tsx:**
- Import `useRecentlyVisited` in ExplorePage
- Call `addVisit` inside a `useEffect` that depends on the current path/segments — when user navigates to a non-root endpoint, record the visit. Use the last segment's label as `name`, construct `path` from the last segment's path, and derive `kind` from the last segment's kind.
- Only record visits for non-root paths (don't record visiting `/_api` itself)

**Update barrel export** in `app/src/hooks/index.ts` to export `useRecentlyVisited` and `RecentlyVisitedItem`.
