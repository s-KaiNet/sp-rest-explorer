    - Port the interfaces from `az-funcs/src/interfaces/` into a single file. Do NOT import from az-funcs — these are new self-contained types for the React app.
    - Interfaces needed: `Metadata`, `EntityType`, `FunctionImport`, `Property`, `NavigationProperty`, `Parameter`
    - Keep the exact same shape as the old Vue app interfaces (the JSON from Azure maps directly to these types):
      - `Metadata`: `{ entities: Record<string, EntityType>; functions: Record<number, FunctionImport> }`
      - `EntityType`: `{ name, fullName, alias?, baseTypeName?, properties: Property[], functionIds: number[], navigationProperties: NavigationProperty[] }`
      - `FunctionImport`: `{ name, isRoot, isComposable, isBindable, returnType: string, parameters: Parameter[], id: number }`
      - `Property`: `{ name, typeName, nullable?: boolean }`
      - `NavigationProperty`: `{ typeName, name }`
      - `Parameter`: `{ name, typeName, nullable: boolean }`
    - Also export `SearchDocument` interface: `{ id: string, name: string, fullName: string, kind: 'entity' | 'function' | 'navProperty', parentEntity?: string }`
    - Also export `ChildEntry` interface: `{ name: string, kind: 'function' | 'navProperty', ref: number | string, returnType?: string }`
    - Also export `LookupMaps` interface: `{ entityByFullName: Map<string, EntityType>, functionById: Map<number, FunctionImport>, entityChildren: Map<string, ChildEntry[]> }`
    - Also export `AppStatus` type: `'idle' | 'loading' | 'ready' | 'error'`

    - `subscribe(callback)` adds listener, returns unsubscribe function
    - `setMetadata(data: Metadata)`: freezes with `Object.freeze()` (shallow — do NOT deep freeze), stores, notifies listeners
    - `getMetadata()`: returns current metadata (for non-React code)
    - `useMetadataSnapshot()`: hook using `useSyncExternalStore(subscribe, getSnapshot, getSnapshot)` — returns `Metadata | null`
    - Import `useSyncExternalStore` from `'react'`

    - Import `AppStatus` from `@/lib/metadata/types`
    - Export `useAppStore`
    - Do NOT store metadata in this store — only status flags

<task type="auto">
  <name>Task 2: Create lookup maps, search index, IndexedDB cache, and boot orchestrator</name>
  <files>

  <action>
    **Step 1: Create lookup maps module** at `app/src/lib/metadata/lookup-maps.ts`:
    - `buildLookupMaps(metadata: Metadata): LookupMaps` function:
      - `entityByFullName`: Map<string, EntityType> — iterate `Object.entries(metadata.entities)`, set fullName → entity
      - `functionById`: Map<number, FunctionImport> — iterate `Object.values(metadata.functions)`, set fn.id → fn. Note: function IDs are sparse (1-3576 with gaps), so use Map not Array.
      - `entityChildren`: Map<string, ChildEntry[]> — for each entity, build array of ChildEntry from navigationProperties (kind: 'navProperty', ref: nav.typeName) and functionIds (kind: 'function', ref: fn.id, returnType: fn.returnType). Sort children: navProperties first, then functions, alphabetically within each group.
    - Module-level `let lookupMaps: LookupMaps | null = null`
    - `initLookupMaps(metadata: Metadata)`: builds and stores maps
    - `getLookupMaps(): LookupMaps | null`: returns current maps (for non-React code)
    - `useLookupMaps(): LookupMaps | null`: hook using `useSyncExternalStore` with same subscribe/getSnapshot pattern as metadata-store (separate listener set)

    **Step 2: Create MiniSearch index module** at `app/src/lib/metadata/search-index.ts`:
    - `buildSearchIndex(metadata: Metadata): MiniSearch<SearchDocument>` function:
      - Create MiniSearch instance with:
        - `fields: ['name', 'fullName']`
        - `storeFields: ['name', 'fullName', 'kind', 'parentEntity']`
        - `searchOptions: { boost: { name: 2 }, fuzzy: 0.2, prefix: true }`
        - `tokenize: (text) => text.split(/[\s._]+/)` — splits "SP.Web" into ["SP", "Web"]
      - Build documents array:
        1. All entities: `{ id: 'entity:{fullName}', name: entity.name, fullName: entity.fullName, kind: 'entity' }`
        2. Root functions: `{ id: 'fn:root:{fn.id}', name: fn.name, fullName: fn.name, kind: 'function' }`
        3. For each entity, its nav properties: `{ id: 'nav:{fullName}/{nav.name}', name: nav.name, fullName: '{entity.name}.{nav.name}', kind: 'navProperty', parentEntity: fullName }`
        4. For each entity, its bound functions: `{ id: 'fn:{fullName}/{fn.id}', name: fn.name, fullName: '{entity.name}.{fn.name}', kind: 'function', parentEntity: fullName }`
      - Use `index.addAll(docs)` (batch add is faster than individual adds)
      - Log index stats to console: `console.log('[SP Explorer] Search index:', docs.length, 'items indexed')`
    - Module-level `let searchIndex: MiniSearch<SearchDocument> | null = null`
    - `initSearchIndex(metadata: Metadata)`: builds and stores index
    - `getSearchIndex(): MiniSearch<SearchDocument> | null`: returns current index

    **Step 3: Create IndexedDB cache module** at `app/src/lib/metadata/metadata-cache.ts`:
    - Use `idb-keyval` (`get`, `set` from 'idb-keyval')
    - Define `CachedMetadata` interface: `{ data: Metadata, timestamp: number }`
    - `getCachedMetadata(): Promise<Metadata | null>`: try `get<CachedMetadata>(CACHE_KEY)`, return `cached.data` if exists, null otherwise. Catch all errors silently (IndexedDB may be unavailable in private browsing).
    - `setCachedMetadata(data: Metadata): Promise<void>`: try `set(CACHE_KEY, { data, timestamp: Date.now() })`. Catch all errors silently.
    - Import CACHE_KEY from constants.

    **Step 4: Create boot orchestrator** at `app/src/lib/metadata/boot.ts`:
    - `bootMetadata()` async function that orchestrates the full loading flow:
      1. Get `setStatus` from `useAppStore.getState()` (access Zustand outside React)
      2. Set status to 'loading'
      3. Try IndexedDB cache first via `getCachedMetadata()`
      4. If cached: call `setMetadata(cached)`, `initLookupMaps(cached)`, `initSearchIndex(cached)`, set status to 'ready'
         - Then start background revalidation: fetch from Azure, if successful call `setCachedMetadata(fresh)` — do NOT update the live singleton mid-session (per research recommendation). Catch errors silently.
      5. If no cache: fetch from METADATA_URL using `fetch()` (not axios — no need for extra dependency), parse JSON, call `setMetadata(data)`, `initLookupMaps(data)`, `initSearchIndex(data)`, `setCachedMetadata(data)`, set status to 'ready'
      6. If fetch fails: set status to 'error' with message "Failed to load API metadata from Azure. Check your connection and try again."
    - `retryBoot()` function: resets status to 'idle', calls `bootMetadata()` again. Exported for the retry button.
    - Log timing to console: `console.log('[SP Explorer] Metadata loaded in', duration, 'ms')`

    **Step 5: Create barrel export** at `app/src/lib/metadata/index.ts`:
    - Re-export types from './types'
    - Re-export `useMetadataSnapshot`, `getMetadata` from './metadata-store'
    - Re-export `useLookupMaps`, `getLookupMaps` from './lookup-maps'
    - Re-export `getSearchIndex` from './search-index'
    - Re-export `bootMetadata`, `retryBoot` from './boot'
    - Do NOT re-export internal functions like `setMetadata`, `initLookupMaps`, `initSearchIndex` — those are used only by boot.ts
