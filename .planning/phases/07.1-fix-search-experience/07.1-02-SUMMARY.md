## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Tree-walk algorithm too restrictive — global entity-visited dedup prevented indexing most paths**
- **Found during:** Post-checkpoint verification (user reported GetByUrlOrAddStub showing few results)
- **Issue:** The global entity-visited Set from Plan 01 meant each entity type was only expanded once across all paths. This produced only ~3.3K unique endpoints — most nested paths were never indexed. For example, `GetByUrlOrAddStub` only appeared at one path instead of the 52+ paths where it's reachable.
- **Fix:** Replaced global entity-visited Set with per-path ancestor tracking (Set passed down each DFS branch) and added depth limit of 4 levels. Index grew from ~3.3K to ~62K endpoints, covering every reachable path.
- **Files modified:** `app/src/lib/metadata/api-tree-walk.ts`, `app/src/lib/metadata/types.ts`
- **Verification:** Search for "GetByUrlOrAddStub" returns 52 results at different paths. Build passes.
- **Committed in:** `acd4a43`

---

**Total deviations:** 1 auto-fixed (1 bug)
**Impact on plan:** Essential for comprehensive search coverage. Without this fix, most nested API paths were invisible to search. No scope creep — addresses the core goal of "find any endpoint at any nesting depth."
