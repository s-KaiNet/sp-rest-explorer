<objective>
Rewrite the CommandPalette component from 3 groups to 2 collapsible groups (Entities, API Endpoints) with "Show more" expansion and fix navigation so entities go to `/entity/<fullName>` and endpoints go to `/_api/<path>`.

Purpose: The current CommandPalette has three groups (entities, functions, nav properties) with a hard cap of 7 per group and broken navigation for most entities (relies on entityPathMap which only covers ~265 root-reachable entities). This rewrite uses the new 2-kind search index from Plan 01, displays results in collapsible groups per CONTEXT.md decisions, and navigates using pre-computed paths stored in each search document — no runtime path resolution needed.

Output: Rewritten `CommandPalette.tsx` with 2-group UI, updated `App.tsx` with simplified selection handling.
</objective>

const handleSelect = useCallback(
  (selection: SearchSelection) => {
    navigate(selection.path)

    // Map search kind to recently-visited kind
    const kindMap: Record<SearchSelection['kind'], 'function' | 'navProperty' | 'root'> = {
      entity: 'root',
