# Phase 07.1 Plan 01: BFS Tree-Walk and Search Index Rewrite Summary

**BFS tree-walk producing ~3,330 endpoint entries with pre-computed _api/ paths, combined with 2,449 entity documents into a two-kind MiniSearch index**

## Accomplishments
- Created BFS tree-walk algorithm that traverses from root functions through entity nav properties and bound functions, building pre-computed `_api/` paths
- Rewrote search index to produce two document kinds: entity (searchable by name + fullName) and endpoint (searchable by leaf name only, path stored but not indexed)
- Updated boot sequence to wire LookupMaps into the search index builder
- Endpoints deduplicated by identity with shortest-path preference; cycle detection via entity-level visited Set

## Decisions Made
- BFS from root functions through nav properties and bound functions — entity-level visited Set for cycle detection, no depth limit needed (tree converges at ~depth 8)
- Endpoint documents set `fullName: ''` so MiniSearch indexes leaf name only — path is stored but not searchable
- Dedup by identity string (`entityFullName|leafName|kind`) with shortest path preference
- Used Option B for boot wiring: call `getLookupMaps()!` after `initLookupMaps(data)` rather than modifying `initLookupMaps` return type

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Fixed pre-existing MiniSearch `limit` option type error**
- **Found during:** Task 2 (build verification)
- **Issue:** `CommandPalette.tsx` passed `{ limit: 21 }` to `searchIndex.search()` but MiniSearch 7.2.0 types don't include `limit` in `SearchOptions`. Pre-existing error only surfaced via `tsc -b` (project build mode), not `tsc --noEmit`.
- **Fix:** Replaced `searchIndex.search(query, { limit: 21 })` with `searchIndex.search(query).slice(0, 21)`
- **Files modified:** `app/src/components/search/CommandPalette.tsx`
- **Verification:** `npm run build` passes
