<task type="auto">
  <name>Task 1: Create GitHub Actions deployment workflow</name>
  <files>.github/workflows/deploy.yml</files>
  <action>
Create `.github/workflows/deploy.yml` with the following structure:

**Trigger:** `on: push: branches: [main]` plus `workflow_dispatch` for manual re-deploys.

**Concurrency:** `concurrency: group: "pages" cancel-in-progress: true` — per user decision, only the latest push deploys.

**Permissions:** `pages: write`, `id-token: write`, `contents: read`.

**Job: build-and-deploy** (single job is fine for this simple pipeline):

Steps:
1. `actions/checkout@v4`
2. `actions/setup-node@v4` with `node-version: '22'` and `cache: 'npm'` with `cache-dependency-path: 'app/package-lock.json'` (per user decision: Node 22, npm caching, subdirectory lock file)
3. `npm ci` with `working-directory: app`
4. `npm run build` with `working-directory: app`
5. `actions/configure-pages@v5`
6. `actions/upload-pages-artifact@v3` with `path: 'app/dist'`
7. `actions/deploy-pages@v4`

**Environment:** Add `environment: name: github-pages, url: ${{ steps.deployment.outputs.page_url }}` and give the deploy step `id: deployment`.

**No path filters** — workflow runs on every push to main (per user decision: Vite builds are fast, avoids edge cases).

Use `defaults: run: working-directory: app` for the npm steps OR per-step working-directory — Claude's discretion per CONTEXT.md.
  </action>
  <verify>
    <automated>node -e "const y=require('fs').readFileSync('.github/workflows/deploy.yml','utf8'); const checks=['push','main','workflow_dispatch','npm ci','npm run build','upload-pages-artifact','deploy-pages','node-version','cancel-in-progress','app/package-lock.json']; const missing=checks.filter(c=>!y.includes(c)); if(missing.length){console.error('MISSING:',missing);process.exit(1)}else{console.log('All checks passed')}"</automated>
    <manual>Review YAML syntax is valid and all required actions are referenced with reasonable versions</manual>
  </verify>
  <done>A complete GitHub Actions workflow file exists at .github/workflows/deploy.yml that triggers on push to main + workflow_dispatch, installs deps with npm ci, builds with npm run build, and deploys via upload-pages-artifact + deploy-pages. Concurrency group cancels in-progress runs.</done>
</task>

<task type="auto">
  <name>Task 2: Update Vite build output and .gitignore</name>
  <files>app/vite.config.ts, .gitignore</files>
  <action>
**app/vite.config.ts:**
- Change `outDir: '../docs'` to `outDir: 'dist'`
- Remove `emptyOutDir: true` — it's the default when outDir is inside project root, and no longer needed since we're not writing outside app/
- Verify `base: '/sp-rest-explorer/'` remains unchanged (BLDG-02 — verification only, do not modify)

**Root .gitignore** (NOT app/.gitignore — per user decision):
- Add `docs/` to prevent any future docs/ from being committed
- Add `app/dist/` to prevent local build output from being committed
- Keep existing entries (`.codenomad/`, `.vite/`)

Note: `app/.gitignore` already has `dist` in it, which covers `app/dist` when git evaluates from within app/. The root `.gitignore` addition of `app/dist/` is belt-and-suspenders per the user's explicit decision.
