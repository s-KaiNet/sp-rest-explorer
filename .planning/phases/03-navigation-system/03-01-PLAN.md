  <action>
Create `useApiNavigation` hook that resolves the current URL into navigation state.

**Input:** React Router's `useParams()` splat from `_api/*` route pattern.

**Output interface:**
```ts
interface BreadcrumbSegment {
  label: string        // Display name (e.g., "web", "Lists", "GetByTitle")
  path: string         // Full route path (e.g., "/_api/web/Lists")
}

interface ApiNavigationState {
  segments: BreadcrumbSegment[]  // Ordered breadcrumb segments
  children: ChildEntry[]         // Children of current node (from lookup maps)
  currentEntity: EntityType | null  // Resolved entity at current path
  isRoot: boolean                // True when at /_api root (no splat)
}
```

**Resolution algorithm:**
1. Get splat from `useParams()["*"]` — this is the path after `_api/` (e.g., `"web/Lists/GetByTitle"`)
2. If no splat or empty, we're at the API root. Return root-level functions (isRoot functions from metadata) as children. Segments = just `[{label: "_api", path: "/_api"}]`.
3. If splat exists, split by `/` to get path parts: `["web", "Lists", "GetByTitle"]`
4. Walk the path parts to build breadcrumb segments:
   - Always start with `{label: "_api", path: "/_api"}`
   - For each part, append `{label: part, path: "/_api/" + parts.slice(0, i+1).join("/")}`
5. Resolve current entity: The first part of the path is always a root function name. Look up the root function by name from the metadata functions (filter where `isRoot === true` and `name === parts[0]`). Get its `returnType`. Look up that entity by fullName in `entityByFullName`. Then for each subsequent path part, find the matching child in `entityChildren` and resolve its target entity:
   - If child.kind === 'navProperty': look up entity by child.ref (which is the typeName/fullName)
   - If child.kind === 'function': look up function by child.ref (ID), get returnType, look up entity by returnType
6. From the resolved current entity, get its children from `entityChildren` map.
7. Return the full `ApiNavigationState`.

**Important edge cases:**
- Path may not fully resolve (e.g., user typed invalid URL) — return what resolves, empty children for unresolved
- Root functions: filter `Object.values(metadata.functions)` where `isRoot === true`, sort alphabetically by name. These become children when at `/_api` root, with `kind: 'function'`
- Use `useLookupMaps()` hook for maps and `useMetadataSnapshot()` for raw metadata access to root functions
- Memoize with `useMemo` keyed on the splat string + maps reference

Also create `app/src/hooks/index.ts` barrel: `export { useApiNavigation } from './use-api-navigation'` and re-export the types `BreadcrumbSegment`, `ApiNavigationState`.
