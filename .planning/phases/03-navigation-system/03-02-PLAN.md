**ResizablePanel component** (`ResizablePanel.tsx`):

Props: `children: React.ReactNode`, `className?: string`

Renders a sidebar container with a draggable right-edge resize handle.

Implementation:
- State: `width` initialized from `localStorage.getItem('sidebar-width')` parsed as number, fallback `280` (default per user decision)
- `isDragging` state (boolean) to track active resize
- On `pointerdown` on the handle div: set `isDragging = true`, capture pointer via `setPointerCapture`
- On `pointermove` (on the handle, captured): calculate new width from `e.clientX`, clamp between `200` (min) and `500` (max) per user decision
- On `pointerup`: set `isDragging = false`, release pointer capture, save width to `localStorage.setItem('sidebar-width', width.toString())`
- Container div: `style={{ width }}` with `shrink-0 flex flex-col h-full relative` classes
- Resize handle: positioned on right edge — `absolute right-0 top-0 bottom-0 w-1 cursor-col-resize hover:bg-primary/20 transition-colors z-10` (per user decision: 1-2px, highlights on hover)
- When dragging: handle gets `bg-primary/30` and body should have `select-none cursor-col-resize` (use `useEffect` to toggle body classes during drag)
- Children rendered inside the container div (the sidebar content fills the space)

**NavigationTransition component** (`NavigationTransition.tsx`):

Two exported components:

`SidebarTransition`: Wraps sidebar content with directional slide animation.
- Props: `children: React.ReactNode`, `pathKey: string`, `direction: 'left' | 'right' | 'none'`
- Uses CSS transitions with a `key` change pattern:
  - Track `prevChildren` and `animationState` in refs/state
  - When `pathKey` changes: set animation class based on direction, after 150ms (per user decision) swap to new children, animate in from opposite side
  - CSS classes: `transition-transform duration-150 ease-out`
  - Going deeper (direction='left'): exit `translate-x-[-100%]`, enter from `translate-x-[100%]`
  - Going back (direction='right'): exit `translate-x-[100%]`, enter from `translate-x-[-100%]`
  - On initial render or direction='none': no animation
- Simpler alternative that's more reliable: Use `key={pathKey}` on a wrapper div with CSS `animate-in` classes from tw-animate-css (already installed). Use `slide-in-from-left` or `slide-in-from-right` based on direction, with `duration-150`. This avoids complex state management.

`ContentTransition`: Wraps content area with fade animation.
- Props: `children: React.ReactNode`, `pathKey: string`
- Simple: wrapper div with `key={pathKey}` and `className="animate-in fade-in duration-150"`
- This uses tw-animate-css (already in the project from Phase 1) for the fade animation on key change

**Determining direction:**
The consuming component (ExplorePage) will compare current depth (segments.length) vs previous depth using a `useRef` to determine direction:
- If current depth > previous: direction = 'left' (going deeper)
- If current depth < previous: direction = 'right' (going back)
- If equal: direction = 'left' (default for same-level navigation)
- This covers browser back/forward: going back increases/decreases depth the same way

**ExplorePage** — Replace the placeholder with the full navigation layout.

```tsx
// Pseudo-structure
function ExplorePage() {
  const navigate = useNavigate()
  const { segments, children, currentEntity, isRoot } = useApiNavigation()

  // Track previous depth for animation direction
  const prevDepthRef = useRef(segments.length)
  const direction = useMemo(() => {
    const prev = prevDepthRef.current
    const curr = segments.length
    if (curr > prev) return 'left' as const
    if (curr < prev) return 'right' as const
    return 'left' as const  // same-level default
  }, [segments.length])

  useEffect(() => {
    prevDepthRef.current = segments.length
  }, [segments.length])

  const pathKey = segments.map(s => s.label).join('/')

  const handleBreadcrumbNavigate = (path: string) => navigate(path)
  const handleSidebarNavigate = (child: ChildEntry) => {
    // Build the navigation path by appending child name to current path
    const currentPath = segments.length > 0
      ? segments[segments.length - 1].path
      : '/_api'
    navigate(`${currentPath}/${child.name}`)
  }

  return (
    <div className="flex flex-1 flex-col">
      {/* Breadcrumb bar — full width above sidebar + content */}
      <BreadcrumbBar segments={segments} onNavigate={handleBreadcrumbNavigate} />

      {/* Sidebar + Content layout */}
      <div className="flex flex-1 overflow-hidden">
        {/* Resizable sidebar */}
        <ResizablePanel>
          <SidebarTransition pathKey={pathKey} direction={direction}>
            <Sidebar children={children} onNavigate={handleSidebarNavigate} />
          </SidebarTransition>
        </ResizablePanel>

        {/* Right border for sidebar separation */}
        {/* (built into ResizablePanel or as adjacent div) */}

        {/* Content area */}
        <div className="flex-1 overflow-y-auto">
          <ContentTransition pathKey={pathKey}>
            <div className="p-6">
              {/* Placeholder content — Phase 4/5 will fill this */}
              {isRoot ? (
                <div className="text-center text-muted-foreground py-12">
                  <p className="text-lg">Select an endpoint from the sidebar to explore</p>
                </div>
              ) : currentEntity ? (
                <div>
                  <h1 className="text-xl font-semibold">{currentEntity.name}</h1>
                  <p className="text-sm text-muted-foreground mt-1">{currentEntity.fullName}</p>
                  {/* Placeholder for entity/function detail — Phase 5 */}
                </div>
              ) : (
                <div className="text-center text-muted-foreground py-12">
                  <p>Endpoint not found</p>
                </div>
              )}
            </div>
          </ContentTransition>
        </div>
      </div>
    </div>
  )
}
```

Key points:
- `children` prop on `Sidebar` may shadow React's children — rename the prop to `items` in Sidebar if needed, or use `entries` as the prop name instead
- The sidebar border-right should be on the ResizablePanel's container: `border-r border-border`
- `overflow-hidden` on the flex container prevents double scrollbars — sidebar scrolls independently (overflow-y-auto on Sidebar), content scrolls independently (overflow-y-auto on content div)
- When at root (`isRoot`), show "Select an endpoint" message in content area
- When entity resolved, show basic entity name/fullName as placeholder (Phase 5 builds real detail)
- When path doesn't resolve, show "Endpoint not found"
