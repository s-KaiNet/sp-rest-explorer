**Dual MiniSearch indexes (name + path) with automatic mode detection — single words search names, slashes/spaces search full REST paths with AND semantics and segment highlighting**

## Accomplishments
- Second MiniSearch index built at boot, tokenized on `/` for path-based endpoint search (~62K entries)
- Automatic search mode detection: single words → name index, queries with `/` or spaces → path index
- CommandPalette routes queries to correct index, hides Entities group in path mode, highlights matching path segments
- Existing name-based search behavior 100% preserved

- `app/src/components/search/CommandPalette.tsx` - HighlightedPath component, mode-aware search routing, conditional Entities group

## Decisions Made
- Path index tokenizes on `/` to match REST path mental model (vs name index which tokenizes on whitespace/dot/underscore)
- AND combinator for multi-term path queries — all tokens must match
- `_api/` alone doesn't trigger path mode (would match all 62K endpoints)
- Path index results skip `kind` filter since they're all endpoints (PathSearchDocument has no `kind` field)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Path index results missing `kind` field — filter returns empty**
- **Found during:** Task 2 (CommandPalette wiring)
- **Issue:** The `entities`/`endpoints` memo filters relied on `r.kind === 'entity'` / `r.kind === 'endpoint'`, but PathSearchDocument has no `kind` field. In path mode, all results would filter to empty arrays.
- **Fix:** In path mode, entities is always `[]` (path index has no entities) and endpoints gets all results unfiltered (path index is endpoints-only).
- **Files modified:** app/src/components/search/CommandPalette.tsx
- **Verification:** Build passes, path mode returns results correctly
- **Committed in:** 7296599 (Task 2 commit)
