<objective>
Add path-based search to the API Endpoints search index so users can find endpoints by their full `_api/...` REST path.

Purpose: SharePoint developers think in REST paths (`_api/web/Lists/GetByTitle`). Currently search only matches leaf names. This adds a second MiniSearch index tokenized on `/` so typing path fragments instantly finds the right endpoint.

Output: Dual-index search system with automatic mode detection — single words search names, slashes or spaces search paths with AND semantics.
</objective>

**3. Add query mode detection function** in `search-index.ts`:

```ts
export type SearchMode = 'name' | 'path'

export function detectSearchMode(query: string): SearchMode {
  // Path mode if query contains a space (AND path search)
  if (query.includes(' ')) return 'path'
  
  // Path mode if query contains / that isn't just "_api/" prefix alone
  if (query.includes('/')) {
    // Strip leading _api/ and check if there's substance after it
    const stripped = query.replace(/^_api\/?$/i, '')
    if (stripped !== '' && stripped !== query.replace(/^_api\/?/i, '')) {
      // Was just "_api/" or "_api" — don't trigger path mode (too broad)
      // Actually: if stripped is empty, that means the whole query was _api/ → name mode
      return 'path'
    }
    // query has / but isn't just _api/ — it has path substance
    if (query.includes('/') && stripped === query) {
      // No _api prefix but has slash — path mode
      return 'path'
    }
    // Just "_api/" alone — don't trigger (too broad, matches all 62K)
    return 'name'
  }
  
  return 'name'
}
```

IMPORTANT: Simplify the above logic. The rule from CONTEXT.md is:
- Contains space → path mode
- Contains `/` → path mode, UNLESS the entire query is just `_api/` or `_api` (no substance after the prefix) → name mode
- Everything else → name mode

So the clean implementation is:
```ts
export function detectSearchMode(query: string): SearchMode {
  if (query.includes(' ')) return 'path'
  if (query.includes('/')) {
    // Don't trigger on bare "_api/" — too broad (matches all 62K endpoints)
    const trimmed = query.replace(/^_api\/?$/i, '')
    return trimmed.length === 0 ? 'name' : 'path'
  }
  return 'name'
}
```

```tsx
function HighlightedPath({ path, query }: { path: string; query: string }) {
  if (!query || query.length < 3) return <span>{path}</span>

  // Tokenize query the same way MiniSearch does: split on / and space
  const queryTokens = query
    .split(/[/ ]+/)
    .filter(Boolean)
    .map(t => t.toLowerCase())

  // Split path into segments on /
  const segments = path.split('/')
  
  return (
    <span>
      {segments.map((segment, i) => {
        const isMatch = queryTokens.some(
          token => segment.toLowerCase().startsWith(token) || segment.toLowerCase().includes(token)
        )
        return (
          <span key={i}>
            {i > 0 && '/'}
            {isMatch ? (
              <strong className="font-bold text-foreground">{segment}</strong>
            ) : (
              <span>{segment}</span>
            )}
          </span>
        )
      })}
    </span>
  )
}
```
